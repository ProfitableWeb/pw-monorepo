# PW-035: Поэтапная анимация появления контента главной страницы

**Статус**: планирование **Приоритет**: средний **Теги**: #animation #ux #framer-motion #homepage #seo

## Проблема

Анимации hero-секции и сетки статей на главной странице запускаются одновременно при монтировании. Визуально контент
появляется хаотично — нет ощущения последовательной «подачи» информации.

## Цель

Поэтапное появление контента: сначала шапка (hero), затем с задержкой — сетка статей. Создать ощущение плавного
раскрытия страницы сверху вниз.

## Требования

1. **Последовательность анимаций**: hero завершает анимацию → запускается анимация сетки статей
2. **Механизм координации**: `onAnimationComplete` (Framer Motion), без `setTimeout`
3. **Stagger карточек**: увеличить интервал между карточками (текущие 15ms → 40-60ms) для читаемости
4. **Accessibility**: сохранить поддержку `prefers-reduced-motion` — контент виден сразу, без ожидания
5. **Skeleton loading**: не требуется (контент рендерится на сервере, SSR)
6. **No-JS фолбэк**: контент видим без JavaScript (`<noscript>` стили или CSS-фолбэк). Серверный HTML не должен скрывать
   контент от ботов и пользователей без JS

## SEO-обоснование

- Контент рендерится на сервере (SSR) — боты получают полный HTML
- `transform` и `opacity` не вызывают Layout Shift (CLS = 0)
- Framer Motion применяет `initial` стили при SSR (inline `opacity: 0`) — без JS-фолбэка контент останется скрытым
- `<noscript>` стили гарантируют видимость для ботов и пользователей без JS

## Верификация

1. Визуально: hero появляется первым, карточки — после завершения hero
2. `prefers-reduced-motion` — анимации отключены, контент виден сразу
3. No-JS: отключить JS в браузере — контент виден
4. `bun turbo type-check --filter=@profitable-web/web` — без ошибок
5. `bun turbo build --filter=@profitable-web/web` — сборка ОК
