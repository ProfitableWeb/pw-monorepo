Создать структурированный раздел документации для хранения архитектурных решений (ADR), стратегий, схем инфраструктуры и
регламентов, которые переживают конкретные задачи и спринты.

`docs/architecture/` и понять как устроена система.

### Формат ADR (Architecture Decision Record)

Каждый ADR следует шаблону:

```markdown
# ADR-NNN: Краткое название решения

## Статус

[Proposed | Accepted | Deprecated | Superseded by ADR-XXX]

## Контекст

Какая проблема решается? Почему это важно?

## Решение

Что мы решили сделать?

## Последствия

- Положительные: ...
- Отрицательные: ...
- Риски: ...

## Альтернативы

Какие варианты рассматривались? Почему отвергнуты?

## Связанные

- PW-XXX — задача реализации
- ADR-YYY — связанное решение
```
