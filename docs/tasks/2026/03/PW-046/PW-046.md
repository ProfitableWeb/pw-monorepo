## Исследование: Яндекс-интеграции

### Яндекс.Метрика — подключение счётчика

Для подключения счётчика нужен только **номер счётчика** (counter ID, например `12345678`). Скрипт `tag.js` загружается
с `https://mc.yandex.ru/metrika/tag.js` и инициализируется:

```javascript
ym(COUNTER_ID, 'init', {
  clickmap: true, // карта кликов
  trackLinks: true, // отслеживание ссылок
  accurateTrackBounce: true, // точный подсчёт отказов
  webvisor: true, // Вебвизор (запись сессий)
});
```

**Для web (Next.js)**: компонент `<Script>` с `strategy="afterInteractive"` + клиентский компонент Metrika. Counter ID
приходит из API (настройки сайта), а не хардкодится.

**Настраиваемые параметры** (выносим в админку):

| Параметр            | Тип    | Описание                               |
| ------------------- | ------ | -------------------------------------- |
| counter_id          | string | Номер счётчика (обязательный)          |
| clickmap            | bool   | Карта кликов                           |
| trackLinks          | bool   | Отслеживание внешних ссылок            |
| accurateTrackBounce | bool   | Точный подсчёт отказов                 |
| webvisor            | bool   | Вебвизор (запись сессий пользователей) |
| trackHash           | bool   | Отслеживание hash в URL                |
| ecommerce           | bool   | Электронная коммерция (не нужно пока)  |

### Яндекс.Метрика API — получение статистики

Для получения данных из Метрики нужен **OAuth-токен** (получается через Яндекс OAuth, permission `metrika:read`).

**Доступные метрики** (что можно показать в дашборде):

| Метрика               | API-параметр                   | Описание          |
| --------------------- | ------------------------------ | ----------------- |
| Визиты                | `ym:s:visits`                  | Количество сессий |
| Просмотры             | `ym:s:pageviews`               | Просмотры страниц |
| Уникальные посетители | `ym:s:users`                   | Уники             |
| Отказы                | `ym:s:bounceRate`              | Процент отказов   |
| Глубина просмотра     | `ym:s:pageDepth`               | Страниц за визит  |
| Время на сайте        | `ym:s:avgVisitDurationSeconds` | Среднее время     |

**Endpoint**: `GET https://api-metrika.yandex.net/stat/v1/data?id={counter_id}&metrics=...&date1=...&date2=...`

### Яндекс Вебмастер API — статус индексации

OAuth-токен с permissions `webmaster:hostinfo`, `webmaster:verify`. Токен действует **6 месяцев**.

**Доступные данные:**

| Endpoint                                                | Данные                           |
| ------------------------------------------------------- | -------------------------------- |
| `GET /v4/user/{uid}/hosts/{hid}/summary`                | SQI, страниц в индексе, проблемы |
| `GET /v4/user/{uid}/hosts/{hid}/indexing/history`       | История индексации по дням       |
| `GET /v4/user/{uid}/hosts/{hid}/sqi-history`            | История изменения SQI            |
| `GET /v4/user/{uid}/hosts/{hid}/search-queries/popular` | Популярные поисковые запросы     |

**Важно**: оба API (Метрика и Вебмастер) используют общий Яндекс OAuth, можно получить один токен с обоими permissions.

### Подключение к Яндекс API — OAuth-попап (принятое решение)

Вместо ручной вставки токена — кнопка «Подключить Яндекс API» с OAuth-попапом:

1. Админ нажимает **«Подключить Яндекс API»**
2. Открывается попап → Яндекс OAuth (`scope=metrika:read+webmaster:hostinfo`, `response_type=code`)
3. Владелец сайта авторизует доступ в Яндексе
4. Яндекс редиректит на callback-страницу с `code`
5. Callback-страница отправляет `code` родительскому окну через `postMessage`
6. Админка отправляет `code` на бэкенд → бэкенд обменивает на токен (`client_id` + `client_secret`) → сохраняет
   зашифрованным
7. Токен **никогда не попадает на фронтенд** — бэкенд проксирует запросы к API

**Нюанс**: это НЕ тот же OAuth, что для входа пользователей. Токен привязан к аккаунту **владельца сайта** (у кого
счётчик Метрики и сайт в Вебмастере), он один на всю систему. Ручная вставка токена — fallback для нестандартных
случаев.

**Для mock-first UI** — карточка подключения показывает два состояния:

- **Не подключено**: кнопка «Подключить Яндекс API» + сворачиваемый fallback «Вставить токен вручную»
- **Подключено**: аккаунт (`ivan@yandex.ru`), права, кнопки «Переподключить» / «Отключить»
