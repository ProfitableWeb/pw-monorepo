# PW-037: Article Workbench — Продвинутый редактор статей

**Статус**: Проектирование / Реализация (Mock-first) **ID задачи**: PW-037 **Контекст**: ProfitableWeb Admin
(`apps/admin`)

## 1. Цель

Создать профессиональную рабочую среду для редактирования статей (Workbench), интегрированную с рабочими пространствами
исследований (Research Workspaces). Поддержка многоформатного редактирования (Markdown/HTML) и визуального контроля.

## 2. Спецификация UI/UX

### Структура табов (Radix UI)

1.  **Вкладка: Мета и SEO**
    - **Поля**: Заголовок, Slug (ЧПУ), Категория, Метки, Краткое описание (Excerpt).
    - **Live Card Preview**: Реальное отображение карточки статьи, как она будет выглядеть в Masonry-сетке на фронтенде.
2.  **Вкладка: Редактор (Workbench)**
    - **Выбор режима**: Выпадающий список (Markdown / HTML / Visual).
    - **Интерфейс Split-View**:
      - **Слева**: Monaco Editor (`@monaco-editor/react`) для кода или разметки.
      - **Справа**: Live Preview с переключателем устройств (Десктоп, Планшет, Мобильный).
    - **AI-гвардеец (Conversion Guard)**: При переключении режимов (например, HTML -> MD), показывать предупреждение:
      «Конвертация требует обработки ИИ для сохранения форматирования. Продолжить?»

### Сайдбар (Контекст и Ревизии)

- **Материалы исследования**: Быстрый доступ к заметкам, источникам и медиа-файлам из связанного Research Workspace
  (PW-039).
- **История ревизий**: Список из последних 5 версий с возможностью восстановления.
- **Статус автосохранения**: Индикатор (Сохранено / Синхронизация / Офлайн).

## 3. Шаги реализации для Агента

1.  **Зависимости**: Добавить `@monaco-editor/react` в `apps/admin`.
2.  **Лейаут**: Создать компонент `ArticleWorkbench` с использованием Radix Tabs и адаптивного контейнера для превью.
3.  **Интеграция Monaco**: Реализовать редактор с подсветкой синтаксиса в зависимости от выбранного режима.
4.  **Device Preview**: Реализовать обертку для правой панели, которая масштабируется под выбранное устройство.
5.  **Состояние**: Связать состояние редактора с контекстом Research Workspace (PW-039), чтобы статьи можно было
    «продвигать» из черновиков.

## 4. Технические ограничения

- Поддержка React 19 / Vite.
- Использование Tailwind CSS для стилизации.
- Использование Lucide для иконок устройств.
- На данном этапе использовать моки для API-запросов (через `lib/api-client.ts`).
