# PW-037: Article Workbench — Продвинутый редактор статей

**Статус**: Реализовано (Mock-first) **ID задачи**: PW-037 **Контекст**: ProfitableWeb Admin (`apps/admin`)

## 1. Цель

Создать профессиональную рабочую среду для редактирования статей (Workbench), интегрированную с рабочими пространствами
исследований (Research Workspaces). Поддержка многоформатного редактирования (HTML/Markdown/Visual) и визуального
контроля.

## 2. Реализованная структура

### 2.1. Вкладки (Radix Tabs)

#### Карточка (`CardTab.tsx`)

Двухколоночный layout с перетаскиваемым разделителем (drag divider):

**Левая колонка** — форма метаданных:

- Заголовок H1 (с подсказками из базы знаний SEO)
- Подзаголовок
- Миниатюра (drag-and-drop загрузка изображений, превью, замена/удаление)
- Краткое описание (Excerpt) — Monaco Editor с двумя режимами (HTML / WYSIWYG)
  - Плавающая панель настроек: тема (13 тем с цветовыми индикаторами), размер шрифта, перенос строк, номера строк,
    миникарта
  - Prettier-форматирование HTML
  - Ресайз высоты редактора (drag handle)
- Категория (select)
- Метки (autocomplete с подсказками, Enter для добавления)
- Кастомный скролл (Radix ScrollArea)

**Правая колонка** — Live Card Preview (`LiveCardPreview.tsx`):

- Iframe-рендер реального компонента `ArticleCard` из `apps/web` через postMessage-протокол
- Переключатель темы (светлая/тёмная) — отправляет `preview:theme` в iframe
- Масштабирование (auto-fit, zoom in/out, reset) с отображением масштаба
- Ручной ввод ширины карточки (input + ограничения min/max)
- Боковые ручки ресайза ширины (зелёные, прозрачные)
- Loading/error overlay с таймаутом 15с и кнопкой «Повторить»

#### SEO (`SeoTab.tsx`)

- SERP-превью (Google) (`SerpPreview.tsx`)
- JSON-LD превью (`JsonLdPreview.tsx`)
- Telegram-превью (`TelegramPreview.tsx`)
- SEO-оценка (`SeoScoreBar.tsx`)
- Счётчики символов (`CharCounter.tsx`)

#### Редактор (`EditorTab.tsx`)

Split-view:

- **Слева**: Monaco Editor (`@monaco-editor/react`) для HTML/Markdown/Visual
  - Выбор режима через dropdown
  - Плавающая перетаскиваемая панель настроек (`EditorSettingsPanel.tsx`):
    - Тема (13 кастомных тем: 8 тёмных + 5 светлых, с цветовыми точками)
    - Размер шрифта (слайдер 10–24px)
    - Размер таба (слайдер 2–8)
    - Перенос строк, номера строк, миникарта (переключатели)
    - Форматтер (Prettier / Monaco встроенный)
    - Ширина строки Prettier
    - HTML whitespace sensitivity
  - Prettier-форматирование (Ctrl+Shift+F)
- **Справа**: Device Preview (`DevicePreview.tsx`)
  - Iframe-рендер реальной страницы статьи из `apps/web`
  - Переключатель устройств (Десктоп 1440px, Планшет 768px, Мобильный 375px)
  - postMessage-протокол (`preview:ready`, `preview:update`)
  - Режим «Полный экран»

#### Артефакты (`ArtifactsTab.tsx`)

Конструктор обогащения контента (post-article blocks):

- **Модуль: Самопроверка**: Список контрольных вопросов с кнопкой «🤖 AI-генерация».
- **Модуль: Источники**: Список ссылок (Название, URL, Тип) с кнопкой «🤖 Импорт из Workspace».
- **Модуль: Глоссарий**: Пары Термин — Определение с кнопкой «🤖 Найти термины».
- **Модуль: Происхождение**: Выбор родительского Research Workspace и тумблер ссылки.
- **Управление**: Каждый блок имеет Switch (Вкл/Выкл).

#### Исследование (`ResearchTab.tsx`)

Связь с родительским Research Workspace (PW-039):

- **Материалы**: Сетка карточек (Заметки, Источники, Медиа).
- **Навигация**: Кнопка «🔗 Перейти к исследованию» для быстрого перехода в воркспейс.

#### История (`HistoryTab.tsx`)

Лог жизненного цикла статьи:

- **Ревизии**: Таймлайн изменений (автор, дата, комментарий).
- **Diff**: Индикатор изменений (аналог WordPress/Git).

### 2.3. Web-роуты для превью

| Роут                  | Файл                                     | Назначение                                 |
| --------------------- | ---------------------------------------- | ------------------------------------------ |
| `/preview`            | `apps/web/src/app/preview/page.tsx`      | Полная страница статьи (для DevicePreview) |
| `/preview/card`       | `apps/web/src/app/preview/card/page.tsx` | Карточка статьи (для LiveCardPreview)      |
| `/preview/layout.tsx` | `apps/web/src/app/preview/layout.tsx`    | Layout с `noindex`, без навигации          |

postMessage-протокол:

- `preview:ready` — iframe сообщает о готовности
- `preview:update` — admin отправляет данные статьи
- `preview:theme` — admin переключает тему (только card preview)
- `preview:resize` — iframe сообщает высоту контента (только card preview)

## 4. Технический стек

- React 19, Vite, Tailwind CSS
- Monaco Editor (`@monaco-editor/react`) — 13 кастомных тем
- Prettier (`prettier/standalone` + `prettier/plugins/html`) — форматирование в браузере
- Radix UI (Tabs, Select, ScrollArea, Switch, Slider, Label)
- Lucide Icons
- postMessage API для iframe-коммуникации
- `react-hook-form` для состояния формы
- Моки данных (`article-mock.ts`)

## 5. Доработки сверх исходного ТЗ

В процессе реализации добавлены улучшения, выявленные при визуальном контроле:

- **Iframe-превью карточки** (вместо inline-имитации) — рендер реального `ArticleCard` из web-приложения
- **Кастомные темы Monaco** — 13 тем (Monokai, Dracula, GitHub Dark/Light, One Dark/Light, Nord/Nord Light, Solarized
  Dark/Light, VS Dark, Light, HC Black) с цветовыми индикаторами
- **Переиспользуемый `EditorSettingsPanel`** — извлечён из EditorTab, используется и в CardTab для excerpt-редактора
- **Переключатель тем превью** — светлая/тёмная тема карточки через postMessage
- **Auto-fit масштабирование** — карточка автоматически вписывается в доступное пространство
- **Боковые ручки ресайза** — изменение ширины карточки перетаскиванием (зелёные, стиль логотипа)
- **Ручной ввод ширины** — числовое поле для точной установки ширины карточки
- **Кастомный скролл** — Radix ScrollArea вместо стандартного скроллбара
- **WYSIWYG-режим excerpt** — альтернатива HTML-редактированию
- **Обработка битых изображений** — многоуровневая: фильтрация пустых URL в admin, проверка в card/page.tsx, onError в
  ArticleCard
- **Стили карточки** — зелёный `<mark>`, исправленные ссылки (border-bottom вместо двойного подчёркивания)
- **SEO Knowledge Base** — интеграция подсказок по полям с базой знаний SEO
