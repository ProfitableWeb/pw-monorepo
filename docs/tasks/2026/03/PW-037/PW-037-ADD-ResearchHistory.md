# PW-037-ADD2: Разделение вкладки «Контекст» на «Исследование» и «История»

**Контекст**: Дополнение к основной задаче PW-037 (Article Workbench) **Статус**: Проектирование / Реализация

## 1. Цель

Разделить текущую вкладку «Контекст» на две самостоятельные логические части: внешние материалы (Исследование) и
внутренний лог изменений (История).

## 2. Спецификация

### Вкладка: Исследование (`ResearchTab.tsx`)

- **Назначение**: Связь с родительским Research Workspace.
- **Содержание**:
  - Сетка (Grid) карточек материалов: Заметки, Источники, Медиа (перенести из старого контекста).
  - **Новый элемент**: Кнопка-ссылка «🔗 Перейти к исследованию» в верхней части вкладки. При клике должен происходить
    переход в соответствующий воркспейс.
- **Стиль**: Кнопка должна быть акцентной (Ghost или Outline с иконкой Link).

### Вкладка: История (`HistoryTab.tsx`)

- **Назначение**: Жизненный цикл статьи и аудит правок.
- **Содержание**:
  - Таймлайн ревизий (перенести из старого контекста).
  - Каждая запись включает: Имя автора, Дату/Время, Краткий комментарий к правке.
- **В будущем**: Здесь появится функционал сравнения версий (Diff).

## 3. Инструкции для Агента

1. **Регистрация табов**: В `ArticleWorkbench.tsx` заменить `'context'` на два новых ключа: `'research'` и `'history'`.
2. **Компоненты**:
   - Создать `ResearchTab.tsx` и `HistoryTab.tsx`.
   - Перенести в них соответствующую логику из старого `WorkbenchSidebar.tsx` или где она была реализована.
3. **Навигация**: Реализовать кнопку перехода к исследованию (пока можно использовать заглушку маршрута, если реальный
   ID исследования еще не прокинут).
4. **Уборка**: Удалить неиспользуемые компоненты или файлы, которые стали избыточными после сплита.
