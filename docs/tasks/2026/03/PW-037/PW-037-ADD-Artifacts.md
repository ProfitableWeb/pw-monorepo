# PW-037-ADD: Вкладка «Артефакты» и конструктор обогащения

**Контекст**: Дополнение к основной задаче PW-037 (Article Workbench) **Статус**: Проектирование / Реализация

## 1. Цель

Реализовать пятую вкладку «Артефакты» (Artifacts), которая служит конструктором дополнительных блоков, идущих после
основного текста статьи. Эти блоки повышают ценность материала (образовательную и доказательную).

## 2. Спецификация компонентов

### Вкладка: Артефакты (`ArtifactsTab.tsx`)

- **Лейаут**: Вертикальный список (Stack) карточек-модулей.
- **Общие элементы модуля**:
  - Заголовок и иконка.
  - Тумблер `Switch` (Вкл/Выкл отображение блока на сайте).
  - Кнопка автоматизации (сиреневый градиент/иконка Sparkles).
- **Стилизация**: Tailwind CSS, Radix UI primitives.

### Модули (Артефакты):

1.  **Самопроверка (Self-Check)**
    - Список пар: `Вопрос` (text) — `Правильный ответ/Подсказка` (textarea).
    - Возможность добавлять/удалять пункты.
    - AI-кнопка: «🤖 Сгенерировать вопросы» (заглушка).

2.  **Источники (Sources)**
    - Таблица или список ссылок.
    - Поля: `Название`, `URL`, `Тип` (Select: Статья, Книга, Видео, Инструмент).
    - AI-кнопка: «🤖 Импорт из Workspace» (заглушка).

3.  **Глоссарий (Glossary)**
    - Список пар: `Термин` (short text) — `Определение` (text).
    - AI-кнопка: «🤖 Найти термины в тексте» (заглушка).

4.  **Происхождение (Provenance)**
    - Выбор связанного `Research Workspace` (Select).
    - Тумблер: «Показывать ссылку на материалы исследования».

## 3. Инструкции для Агента

1.  **Добавить вкладку**: Зарегистрировать `'artifacts'` в списке табов `ArticleWorkbench.tsx`.
2.  **Создать `ArtifactsTab.tsx`**: Реализовать интерфейс согласно описанию выше.
3.  **State**: Добавить соответствующие поля в `ArticleFormData` и `mockArticle`.
4.  **UI-Kit**: Использовать `Accordion` (Radix) для сворачивания/разворачивания модулей, чтобы не перегружать экран.
5.  **AI-Buttons**: Реализовать кнопки с анимацией «активного ожидания» (loading state) при нажатии (только визуально).
