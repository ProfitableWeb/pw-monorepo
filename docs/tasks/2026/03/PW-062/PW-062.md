## Постановка задачи

### Описание

Блоки кода (`<pre><code>`) в статьях отображаются без подсветки синтаксиса — просто моноширинный текст на сером фоне.
Нужно добавить полноценную подсветку синтаксиса с поддержкой light/dark тем.

### Решение — Shiki

[Shiki](https://shiki.style/) — библиотека подсветки синтаксиса, использующая грамматики TextMate (те же, что в VS
Code).

**Почему Shiki:**

- Рендерит статический HTML на сервере — zero client JS для подсветки
- Точная подсветка как в VS Code (те же грамматики)
- Нативная поддержка dual themes (light + dark) через CSS-переменные
- Идеально для Next.js SSR/RSC
- Активно поддерживается, используется в документации Next.js, Vite, Astro

**Альтернативы (отвергнуты):**

- Monaco Editor — полноценный редактор (~2-3MB), избыточен для read-only подсветки
- Prism.js — легковесный, но менее точные грамматики, требует клиентский JS
- highlight.js — аналогично Prism, менее точен

### Цель

Блоки кода в статьях отображаются с подсветкой синтаксиса, автоматически переключающейся между светлой и тёмной темой.

### Критерии приемки

- [ ] Блоки `<pre><code>` в статьях рендерятся с подсветкой синтаксиса
- [ ] Подсветка работает для основных языков: JavaScript/TypeScript, Python, HTML, CSS, JSON, Bash, SQL
- [ ] Поддержка light/dark тем (переключение без перезагрузки)
- [ ] Подсветка рендерится на сервере (SSR) — нет layout shift
- [ ] Стили блоков кода консистентны с текущей типографикой статей (`_shared-typography.scss`)
- [ ] Копирование кода из блока работает корректно

### Зависимости

- Shiki v2+ (`bun add shiki` в apps/web)
- Текущая система тем: `styles/themes/{light,dark}/`
- Компоненты: `ArticleContent.tsx`, `ArticleContentOneColumn.tsx`
- Миксины: `_shared-typography.scss` (`article-pre`, `article-code`)

### Предложенное решение

#### Подход A: Трансформация HTML на сервере (рекомендуется)

Статьи хранятся как HTML в БД. Подсветку можно применять при рендере на сервере:

1. **Создать утилиту `lib/code-highlight.ts`** — серверная функция, которая парсит HTML статьи, находит блоки
   `<pre><code class="language-xxx">` и заменяет их на HTML с подсветкой от Shiki
2. **Dual themes** — использовать `shiki.createHighlighter()` с двумя темами, Shiki генерирует CSS-переменные для
   переключения
3. **Интеграция** — вызывать трансформацию в серверных компонентах (RSC) или в `getArticleBySlug` до передачи в
   клиентский `ArticleContent`
4. **Стили** — обновить `article-pre` миксин для стилизации Shiki-блоков

#### Подход B: Клиентский хайлайтинг (запасной)

Если серверная трансформация HTML окажется сложной из-за DOMPurify:

1. Использовать `shiki/bundle/web` (облегчённый бандл)
2. Подсветка в `useEffect` после DOMPurify-санитизации
3. Минус: layout shift при загрузке

#### Шаги реализации

1. `bun --cwd apps/web add shiki`
2. Создать `lib/code-highlight.ts` с Shiki highlighter (singleton, lazy init)
3. Выбрать темы: light (например `github-light` или `one-light`) + dark (`github-dark` или `one-dark-pro`)
4. Трансформировать HTML блоков кода перед передачей в ArticleContent
5. Обновить `_shared-typography.scss` — стили для Shiki-блоков (padding, border-radius, scrollbar)
6. Убедиться что DOMPurify не стрипает Shiki-атрибуты (`style`, `span` с `color`)
7. Добавить кнопку «Скопировать» для блоков кода (опционально)
8. Проверить на реальных статьях с кодом
