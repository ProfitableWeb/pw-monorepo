## Постановка задачи

### Описание

Поиск и фильтрация статей в админке работают **клиентски** — `filterArticles()` фильтрует только загруженные статьи
(первая страница, limit=20). Если статья не попала в первые 20 — поиск её не найдёт. Счётчики статистики тоже показывают
длину загруженного массива, а не реальное количество в БД.

### Причина

Backend API (`GET /api/admin/articles`) уже поддерживает серверные параметры: `search`, `status`, `category`, `page`,
`limit`, `sort_by`, `order`. Фронтенд просто не передаёт их.

### Критерии приемки

- [x] Поиск по названию отправляется как `search` параметр в API
- [x] Фильтр по статусу отправляется как `status` параметр в API
- [x] Фильтр по категории отправляется как `category` параметр в API
- [x] Статистика (всего, опубликовано, черновиков) использует серверный `/stats` эндпоинт
- [x] Убрана клиентская фильтрация `filterArticles()`
- [x] Все 38 статей доступны через поиск

## План реализации

1. `ArticlesSection.tsx` — передавать search/status/category в `useAdminArticles(params)`
2. `useAdminArticles.ts` — убедиться что params прокидываются (уже работает)
3. `articles.utils.ts` — удалить `filterArticles()` (клиентская фильтрация больше не нужна)
4. Статистика — использовать `meta.total` вместо `articles.length`
5. Backend — добавить отдельный эндпоинт или поля для счётчиков по статусам
