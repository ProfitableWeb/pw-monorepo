# PW-047: Backend — модель SEO-настроек и Admin API

**Статус**: DONE **Создано**: 2026-03-16 **Приоритет**: High **Компонент**: API + Admin

**Зависимости**: PW-046 (mock-first UI — DONE)

## Проблема

Mock-first UI для SEO-инструментов реализован (PW-046), но все данные — из констант в `seo.constants.ts`. Нужен бэкенд
для хранения и управления SEO-настройками: конфигурация sitemap, robots.txt, RSS-фид, мета-директивы по умолчанию и
параметры счётчика Яндекс.Метрики.

## Цель

Создать бэкенд-слой (модель → сервис → API) для хранения SEO-настроек и подключить к нему существующий UI админки.

---

## Архитектурное решение: расширение SystemSettings

`SystemSettings` — singleton-модель (одна строка в таблице). Сейчас хранит только `timezone`. Добавляем JSONB-поля для
SEO-конфигурации. Это проще и целостнее, чем создавать отдельную таблицу — все системные настройки в одном месте.

### Новые поля в SystemSettings

```python
# models/system_settings.py — добавить:
sitemap_config = Column(JSONB, nullable=False, server_default='{}')
robots_txt = Column(Text, nullable=False, server_default='')
rss_config = Column(JSONB, nullable=False, server_default='{}')
default_meta_directives = Column(JSONB, nullable=False, server_default='{}')
metrika_config = Column(JSONB, nullable=False, server_default='{}')
```

### Структура JSONB-полей

```python
# sitemap_config:
{
    "enabled": true,
    "include_articles": true,
    "include_categories": true,
    "include_tags": false,
    "include_static_pages": true,
    "priorities": {"home": 1.0, "articles": 0.8, "categories": 0.6, "tags": 0.4, "static_pages": 0.5},
    "changefreq": {"home": "daily", "articles": "weekly", "categories": "weekly", "tags": "monthly", "static_pages": "monthly"}
}

# rss_config:
{
    "enabled": true,
    "format": "atom",  # "rss2" | "atom"
    "item_count": 20,
    "content_mode": "excerpt",  # "full" | "excerpt"
    "include_articles": true,
    "include_category_updates": false
}

# default_meta_directives:
{
    "articles": {"index": true, "follow": true, "noarchive": false},
    "categories": {"index": true, "follow": true, "noarchive": false},
    "tags": {"index": false, "follow": true, "noarchive": false}
}

# metrika_config:
{
    "counter_id": "",
    "clickmap": true,
    "track_links": true,
    "accurate_track_bounce": true,
    "webvisor": true,
    "track_hash": false
}
```

def get_seo_settings(db: Session) -> dict:

**Схемы** (`schemas/system_settings.py` — расширить):

```python
class SitemapConfigSchema(BaseModel):
    enabled: bool = True
    include_articles: bool = True
    include_categories: bool = True
    include_tags: bool = False
    include_static_pages: bool = True
    priorities: dict[str, float] = {}
    changefreq: dict[str, str] = {}

class RssConfigSchema(BaseModel):
    enabled: bool = True
    format: Literal["rss2", "atom"] = "atom"
    item_count: int = 20
    content_mode: Literal["full", "excerpt"] = "excerpt"
    include_articles: bool = True
    include_category_updates: bool = False

class MetaDirectivesSchema(BaseModel):
    index: bool = True
    follow: bool = True
    noarchive: bool = False

class MetrikaConfigSchema(BaseModel):
    counter_id: str = ""
    clickmap: bool = True
    track_links: bool = True
    accurate_track_bounce: bool = True
    webvisor: bool = True
    track_hash: bool = False

class SeoSettingsResponse(BaseModel):
    sitemap_config: SitemapConfigSchema
    robots_txt: str
    rss_config: RssConfigSchema
    default_meta_directives: dict[str, MetaDirectivesSchema]
    metrika_config: MetrikaConfigSchema

class SeoSettingsUpdateRequest(BaseModel):
    sitemap_config: SitemapConfigSchema | None = None
    robots_txt: str | None = None
    rss_config: RssConfigSchema | None = None
    default_meta_directives: dict[str, MetaDirectivesSchema] | None = None
    metrika_config: MetrikaConfigSchema | None = None
```

## Публичный эндпоинт для web

Помимо admin-эндпоинтов, нужен **публичный** (без авторизации) для генерации sitemap/robots/RSS на стороне web:

```
GET /api/seo/config → { sitemap_config, robots_txt, rss_config, metrika_config }
```

Без `default_meta_directives` (они нужны только админке). `metrika_config.counter_id` нужен web для вставки скрипта.
