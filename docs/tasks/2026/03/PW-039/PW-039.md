# PW-039: Рефакторинг backend — доменные пакеты внутри слоёв

**Статус**: Выполнено **ID задачи**: PW-039 **Контекст**: ProfitableWeb API (`apps/api`) **Зависит от**: PW-038 (Article
CRUD API)

## 1. Цель

Перейти от плоской структуры `services/` и `schemas/` к доменным пакетам, когда домен разрастается до 3+ файлов.
Подготовить кодовую базу к масштабированию (AI-агент, новые домены).

## 2. Проблема

После PW-038 домен «статьи» расползся по 6 файлам с prefix-именованием:

```
services/
├── article.py              # публичные запросы
├── admin_article.py        # admin CRUD
├── article_revision.py     # ревизии
├── reading_time.py         # утилита
├── slug.py                 # общая утилита
└── ...

schemas/
├── article.py              # публичные схемы
├── admin_article.py        # admin схемы
└── ...
```

С добавлением AI-агента `services/` разрастётся до 15-20 файлов. Плоская структура перестаёт масштабироваться.

## 3. Принятый подход

**Доменные пакеты внутри слоёв**:

- 1-2 файла в домене → плоский файл
- 3+ файлов → пакет с `__init__.py`
- Утилиты, используемые несколькими доменами, остаются в корне слоя
- `models/` — всегда плоский (Alembic auto-discovery через `__init__.py`)

## 4. Карта перемещений

### services/ — группировка домена «статьи»

| Было                           | Стало                               | Причина                            |
| ------------------------------ | ----------------------------------- | ---------------------------------- |
| `services/article.py`          | `services/articles/queries.py`      | Публичные запросы (PUBLISHED only) |
| `services/admin_article.py`    | `services/articles/admin.py`        | Admin CRUD                         |
| `services/article_revision.py` | `services/articles/revisions.py`    | Ревизии контента                   |
| `services/reading_time.py`     | `services/articles/reading_time.py` | Утилита только для статей          |
| `services/slug.py`             | **остаётся** `services/slug.py`     | Используется и для тегов           |

### schemas/ — группировка домена «статьи»

| Было                       | Стало                        |
| -------------------------- | ---------------------------- |
| `schemas/article.py`       | `schemas/articles/public.py` |
| `schemas/admin_article.py` | `schemas/articles/admin.py`  |

### Без изменений

- `services/user.py`, `services/comment.py`, `services/storage.py`, `services/avatar.py`, `services/system_settings.py`
- `schemas/common.py`, `schemas/auth.py`, `schemas/user.py`, `schemas/comment.py`, `schemas/category.py`
- `schemas/admin_tag.py`, `schemas/admin_category.py`, `schemas/admin_settings.py`
- `models/` — весь каталог
- `api/` — уже организован с `admin/` подпакетом
- `auth/` — уже пакет с `oauth/`

## 5. Обновлённые импорты

### Потребители services

```python
# api/articles.py, api/categories.py
# Было:  from src.services import article as article_service
# Стало: from src.services.articles import queries as article_service

# api/admin/articles.py
# Было:  from src.services import admin_article as admin_service
# Стало: from src.services.articles import admin as admin_service

# api/admin/articles.py
# Было:  from src.services import article_revision as revision_service
# Стало: from src.services.articles import revisions as revision_service
```

### Внутренние импорты services/articles/admin.py

```python
# Было:
#   from src.services import article_revision as revision_service
#   from src.services.reading_time import calculate_reading_time
# Стало:
#   from src.services.articles import revisions as revision_service
#   from src.services.articles.reading_time import calculate_reading_time
#   from src.services.slug import ensure_unique_slug, generate_slug  ← без изменений
```

### Потребители schemas

```python
# api/articles.py, api/categories.py
# Было:  from src.schemas.article import ArticleListItem, ArticleResponse
# Стало: from src.schemas.articles.public import ArticleListItem, ArticleResponse

# api/admin/articles.py
# Было:  from src.schemas.admin_article import (...)
# Стало: from src.schemas.articles.admin import (...)
```

## 6. Результат

```
apps/api/src/
├── services/
│   ├── articles/               # Доменный пакет
│   │   ├── __init__.py         # Пустой
│   │   ├── queries.py          # Публичные запросы (PUBLISHED)
│   │   ├── admin.py            # Admin CRUD
│   │   ├── revisions.py        # Ревизии
│   │   └── reading_time.py     # Расчёт времени чтения
│   ├── slug.py                 # Общая утилита (статьи + теги)
│   ├── user.py                 # Плоский — 1 файл
│   ├── comment.py
│   └── ...
├── schemas/
│   ├── articles/               # Доменный пакет
│   │   ├── __init__.py         # Пустой
│   │   ├── public.py           # ArticleResponse, ArticleListItem
│   │   └── admin.py            # ArticleCreateRequest, ArticleAdminResponse, ...
│   ├── common.py               # Плоский
│   ├── category.py
│   └── ...
└── models/                     # Всегда плоский
    ├── __init__.py
    ├── article.py
    └── ...
```
