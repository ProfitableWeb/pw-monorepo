## Постановка задачи

### Описание

Данные автора на странице статьи полностью захардкожены в двух местах:

- **`AuthorCard`** (сайдбар) — имя, описание, аватар, соцсети статичны
- **`ArticleAuthorBlock`** (шапка статьи) — использует `AUTHOR_DATA` из `config/author.ts`

API отдаёт только `author: "Николай Егоров"` (строку). Нужно расширить модель пользователя, API и фронт, чтобы все
компоненты автора заполнялись из данных привязанного к статье пользователя.

### Текущее состояние

- **Модель `User`**: `name`, `email`, `avatar`, `role` — нет `bio` и `social_links`
- **API `ArticleResponse`**: `author: str | None` — только имя строкой
- **API `AuthorBrief`** (админка): уже объект `{id, name}` — нужно расширить
- **Фронт `AuthorCard`** (сайдбар): хардкод имени, описания, аватара, соцсетей
- **Фронт `ArticleAuthorBlock`** (шапка): хардкод из `config/author.ts` (`AUTHOR_DATA`)
- **SEO `generateArticleJsonLd()`**: захардкоженный `AUTHOR_SCHEMA` из `config/author.ts`
- **MCP-тулы**: `mcp/tools/articles.py` сериализует автора как строку

### Цель

Карточка автора на странице статьи отображает реальные данные пользователя: имя, аватар, описание и соцсети из БД.
JSON-LD разметка генерирует динамический author schema. Данные автора управляются через админку и настройки профиля.

### Критерии приемки

- [ ] Модель `User` расширена полями `bio` (Text) и `social_links` (JSONB)
- [ ] Миграция Alembic создана и применяется
- [ ] API отдаёт автора статьи как объект (id, name, avatar, bio, socialLinks) вместо строки
- [ ] `AuthorCard` (сайдбар) отображает данные из API
- [ ] `ArticleAuthorBlock` (шапка) отображает данные из API
- [ ] Аватар подтягивается из `user.avatar` (с fallback на дефолтное изображение)
- [ ] Соцсети отображаются динамически из `social_links`
- [ ] Если соцсети не указаны — блок иконок не отображается
- [ ] JSON-LD `author` schema генерируется динамически из данных автора
- [ ] MCP-тулы сериализуют автора как объект (синхронно с API)
- [ ] Админка: управление bio и social_links для любого пользователя
- [ ] Сайт: пользователь может редактировать bio и social_links в настройках профиля
- [ ] Обратная совместимость: статьи без автора — fallback на `config/author.ts`

## Техническое решение

### 1. Backend — модель User

```python
# apps/api/src/models/user.py
from sqlalchemy import Text
from sqlalchemy.dialects.postgresql import JSONB

bio: Mapped[str | None] = mapped_column(Text)
social_links: Mapped[dict | None] = mapped_column(JSONB)
```

Формат `social_links`:

```json
{
  "vk": "https://vk.com/...",
  "telegram": "https://t.me/...",
  "github": "https://github.com/...",
  "dzen": "https://dzen.ru/...",
  "website": "https://..."
}
```

### 2. Backend — миграция Alembic

```bash
cd apps/api
uv run alembic revision --autogenerate -m "add bio and social_links to users"
uv run alembic upgrade head
```

### 3. Backend — схемы и API

Новая схема `AuthorProfile` для публичного API:

```python
# apps/api/src/schemas/articles/public.py
class AuthorProfile(BaseModel):
    id: str
    name: str
    avatar: str | None = None
    bio: str | None = None
    social_links: dict[str, str] | None = None

class ArticleResponse(BaseModel):
    ...
    author: AuthorProfile | None = None  # было: str | None
```

Расширить `AuthorBrief` в админских схемах:

```python
# apps/api/src/schemas/articles/admin.py
class AuthorBrief(BaseModel):
    id: str
    name: str
    avatar: str | None = None  # добавить
```

Сериализация в `api/articles.py` (`_article_to_response()`):

```python
author=AuthorProfile(
    id=str(article.author.id),
    name=article.author.name,
    avatar=article.author.avatar,
    bio=article.author.bio,
    social_links=article.author.social_links,

```

```typescript
// было: всегда AUTHOR_SCHEMA
// стало: динамический author из article.author, fallback на AUTHOR_SCHEMA
const authorSchema = article.author
```

### 8. Frontend — поведение соцсетей

- Если `social_links` пустой/null — блок иконок соцсетей **не отображается** в карточке автора
- Иконки рендерятся только для заполненных ссылок (например, если указан только VK и Telegram — покажутся только они)
- Известные платформы с иконками в `SocialIcons`: `vk`, `telegram`, `dzen`, `github`
- Произвольные ссылки (portfolio, blog, любой URL) — иконка глобуса как универсальный fallback
