# PW-049: Web — генерация sitemap.xml, robots.txt, RSS и подключение счётчика Метрики

**Статус**: TODO **Создано**: 2026-03-16 **Приоритет**: High **Компонент**: Web (Next.js)

**Зависимости**: PW-047 (SEO-настройки API), частично PW-048 (counter_id из настроек)

## Проблема

Сейчас `profitableweb.ru` не имеет:

- `/sitemap.xml` — поисковые системы не знают структуру сайта
- `/robots.txt` — нет явных правил для роботов
- `/rss.xml` или `/feed` — нет RSS-подписки для читателей
- Счётчик Яндекс.Метрики — нет отслеживания посещаемости

Все эти файлы должны генерироваться динамически на основе настроек из API (PW-047).

## Цель

Реализовать серверную генерацию sitemap.xml, robots.txt, RSS/Atom-фида в Next.js и подключить скрипт Яндекс.Метрики.

### PW-049-A: Sitemap.xml

**Файл**: `apps/web/src/app/sitemap.ts`

Next.js 15 поддерживает
[Metadata API для Sitemap](https://nextjs.org/docs/app/api-reference/file-conventions/metadata/sitemap):

```typescript
import type { MetadataRoute } from 'next';

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  // 1. Fetch SEO config from API: GET /api/seo/config → sitemap_config
  // 2. Fetch published articles: GET /api/articles?limit=1000
  // 3. Fetch categories: GET /api/categories
  // 4. Build sitemap entries based on config:

  const entries: MetadataRoute.Sitemap = [];

  // Главная
  entries.push({
    url: `${baseUrl}/`,
    lastModified: new Date(),
    changeFrequency: config.changefreq.home,
    priority: config.priorities.home,
  });

  // Статьи (если include_articles)
  if (config.include_articles) {
    articles.forEach(article => {
      entries.push({
        url: `${baseUrl}/${article.slug}`,
        lastModified: article.updated_at,
        changeFrequency: config.changefreq.articles,
        priority: config.priorities.articles,
      });
    });
  }

  // Категории, теги, статические страницы — аналогично

  return entries;
}
```

**Кэширование**: Next.js кэширует sitemap по умолчанию. Добавить `revalidate` через `generateStaticParams` или
`export const revalidate = 3600` (1 час).

### PW-049-B: Robots.txt

**Файл**: `apps/web/src/app/robots.ts`

```typescript
import type { MetadataRoute } from 'next';

export default async function robots(): Promise<MetadataRoute.Robots> {
  // Fetch from API: GET /api/seo/config → robots_txt
  // Два варианта:
  // A) Вернуть как структурированный объект (Next.js генерирует)
  // B) Вернуть как Route Handler с text/plain (полный контроль)

  return {
    rules: [
      {
        userAgent: '*',
        allow: '/',
        disallow: ['/admin/', '/api/', '/preview/'],
      },
    ],
    sitemap: `${baseUrl}/sitemap.xml`,
    host: 'profitableweb.ru',
  };
}
```

**Альтернатива** (если нужен полный контроль над текстом): `app/robots.txt/route.ts` как Route Handler, возвращающий
`robots_txt` из API как `text/plain`.

**Рекомендация**: Route Handler, т.к. в админке robots.txt редактируется как текст, а не как структура.

```typescript
// app/robots.txt/route.ts
export async function GET() {
  const config = await fetch(`${API_URL}/api/seo/config`);
  const { robots_txt } = await config.json();
  return new Response(robots_txt, {
    headers: { 'Content-Type': 'text/plain' },
  });
}
```

### PW-049-C: RSS/Atom-фид

**Файл**: `apps/web/src/app/feed.xml/route.ts` (или `rss.xml/route.ts`)

```typescript
export async function GET() {
  // 1. Fetch RSS config from API: GET /api/seo/config → rss_config
  // 2. Fetch published articles: GET /api/articles?limit={rss_config.item_count}
  // 3. Generate feed based on config.format ("atom" | "rss2")

  if (config.format === 'atom') {
    return new Response(generateAtomFeed(articles, config), {
      headers: { 'Content-Type': 'application/atom+xml; charset=utf-8' },
    });
  } else {
    return new Response(generateRss2Feed(articles, config), {
      headers: { 'Content-Type': 'application/rss+xml; charset=utf-8' },
    });
  }
}
```

**Генерация XML**: использовать библиотеку `feed` (npm) или генерировать вручную (шаблонные строки). Рекомендация:
библиотека `feed` — поддерживает и Atom, и RSS 2.0.

**Содержимое записей** (по `content_mode`):

- `"full"` → полный HTML контент статьи
- `"excerpt"` → `article.excerpt` (первые 200 символов)

**Ссылка в `<head>` layout.tsx:**

```html
<link rel="alternate" type="application/atom+xml" title="ProfitableWeb RSS" href="/feed.xml" />
```

## Критерии приёмки

- [ ] `GET /sitemap.xml` возвращает валидный XML со всеми опубликованными статьями
- [ ] Sitemap учитывает настройки: включение/исключение типов контента, приоритеты, частоту
- [ ] `GET /robots.txt` возвращает текст из настроек
- [ ] `GET /feed.xml` возвращает валидный Atom/RSS 2.0 фид
- [ ] RSS содержит корректное количество записей и режим содержимого (full/excerpt)
- [ ] `<link rel="alternate">` в `<head>` указывает на фид
- [ ] Счётчик Метрики подключается при наличии counter_id в настройках
- [ ] При пустом counter_id скрипт не загружается
- [ ] Все параметры Метрики (clickmap, webvisor, etc.) передаются из настроек
- [ ] `bun turbo build` проходит без ошибок
