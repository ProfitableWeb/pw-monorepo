# PW-051: Категории — CRUD + API-интеграция админки

**Статус**: TODO **Создано**: 2026-03-17 **Приоритет**: Normal **Компонент**: 🐍 API + ⚙️ Admin

**Зависимости**: PW-027 (модель Category), PW-029 (интеграция админки с API)

## Проблема

Раздел «Категории» в админке (`sections/categories/CategoriesSection.tsx`) работает в режиме read-only + local state:

- Категории **загружаются** из API (`GET /categories`) — это работает
- Создание, редактирование, удаление — только в локальном state, теряются при перезагрузке
- DnD-сортировка и вложенность (parentId) — тоже локальные, не сохраняются
- В модели `Category` нет полей `parent_id` и `order` (UI их поддерживает)
- Бэкенд не имеет admin-эндпоинтов для мутаций (только GET)

## Цель

Полностью рабочий CRUD категорий через API. Сохранение порядка и вложенности. Изменения персистентны.

## PW-051-A: Backend

### Миграция

Добавить в модель `Category`:

```python
parent_id: Mapped[UUID | None] = mapped_column(ForeignKey("categories.id"), nullable=True)
order: Mapped[int] = mapped_column(default=0)

parent: Mapped["Category | None"] = relationship(remote_side="Category.id")
children: Mapped[list["Category"]] = relationship(back_populates="parent")
```

### Схемы

```python
# schemas/admin_category.py
class CategoryCreateRequest(BaseModel):
    name: str
    slug: str | None = None        # Авто-генерация из name если не передан
    subtitle: str | None = None
    description: str | None = None
    icon: str | None = None
    color: str | None = None       # hex, e.g. "#ef4444"
    parent_id: UUID | None = None
    order: int = 0

class CategoryUpdateRequest(BaseModel):
    name: str | None = None
    slug: str | None = None
    subtitle: str | None = None
    description: str | None = None
    icon: str | None = None
    color: str | None = None
    parent_id: UUID | None = None
    order: int | None = None

class CategoryReorderRequest(BaseModel):
    """Массовое обновление порядка и вложенности."""
    items: list[CategoryOrderItem]

class CategoryOrderItem(BaseModel):
    id: UUID
    parent_id: UUID | None = None
    order: int
```

### Эндпоинты (`/admin/categories`)

| Метод  | Путь                      | Описание                          |
| ------ | ------------------------- | --------------------------------- |
| GET    | `/admin/categories`       | Список (уже есть)                 |
| POST   | `/admin/categories`       | Создать категорию                 |
| PUT    | `/admin/categories/{id}`  | Обновить категорию                |
| DELETE | `/admin/categories/{id}`  | Удалить (если нет статей)         |
| PUT    | `/admin/categories/order` | Массовое обновление порядка (DnD) |

### Сервис (`services/category.py`)

- `create_category(db, data)` — создание + авто-slug
- `update_category(db, id, data)` — частичное обновление
- `delete_category(db, id)` — удаление (проверка: нет привязанных статей)
- `reorder_categories(db, items)` — массовое обновление order + parent_id

## Критерии приёмки

- [ ] Создание категории сохраняется в БД
- [ ] Редактирование категории (имя, slug, цвет, описание) сохраняется
- [ ] Удаление категории работает (ошибка если есть статьи)
- [ ] DnD-сортировка сохраняет порядок в БД
- [ ] DnD-вложенность (перетащить в дочернюю) сохраняет parent_id
- [ ] При перезагрузке страницы данные не теряются
- [ ] Build + lint + type-check проходят

## Технические решения

- **slug**: авто-генерация через существующий `services/slug.py` (транслитерация)
- **delete**: запрет удаления если `article_count > 0` (409 Conflict)
- **order**: `PUT /admin/categories/order` принимает массив `{id, parent_id, order}[]` — один запрос после DnD
- **вложенность**: максимум 1 уровень (как в UI) — parent_id != null только у дочерних
