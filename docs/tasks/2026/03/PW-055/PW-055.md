# PW-055: Исправление iframe-предпросмотра статей в админке

**Статус**: DONE **Создано**: 2026-03-18 **Завершено**: 2026-03-18 **Приоритет**: High **Компонент**: ⚙️ Admin + ⚛️ Web

## Проблема

Iframe-предпросмотр статей в редакторе админки не работал ни в dev, ни в production. Несколько независимых багов:

1. **WEB_URL на проде** — захардкожен `localhost:3000`, iframe не загружался
2. **«new» как articleId** — переход в `/admin/content/articles/new` отправлял GET-запрос с `id=new` → 400 Bad Request
3. **ALLOWED_ORIGINS** — preview-страница web отклоняла postMessage с origin продакшена
4. **Контент не попадал в превью** — `EditorTab` читал `formData.content` из react-hook-form (стейл), а не актуальный из
   Zustand-стора
5. **Моковые артефакты** — preview-страница рендерила захардкоженные `exampleSelfAssessmentQuestions` вместо реальных
   данных
6. **Артефакты не передавались** — `PreviewArticleData` не включал `artifacts`, данные из вкладки «Артефакты» не
   доходили до превью

## Решения

### 1. Умный fallback WEB_URL ✅

`preview.types.ts` — централизованный `WEB_URL` с приоритетами:

```
VITE_WEB_URL → (dev: localhost:3000) → (prod: window.location.origin)
```

Все файлы импортируют из `preview.types.ts` вместо собственных определений.

### 2. Фильтрация «new» как articleId ✅

`App.tsx` — при навигации к редактору проверяется `route.params.id === 'new'` → `undefined`.

### 3. ALLOWED_ORIGINS с динамическим origin ✅

`preview/page.tsx` и `preview/card/page.tsx` — добавлен `window.location.origin` в список разрешённых origins. На проде
admin и web живут на одном домене (через nginx), поэтому origin совпадает.

### 4. Актуальный content из Zustand ✅

`EditorTab.tsx` — `toPreviewData({ ...formData, content })` — `content` берётся из `useArticleEditorStore`, а не из
react-hook-form. Zustand обновляется при каждом нажатии клавиши в Monaco, react-hook-form — с задержкой.

### 5. Удаление моковых данных ✅

`preview/page.tsx` — удалены все захардкоженные моки (`exampleSelfAssessmentQuestions`, `exampleArticleResources`,
`MOCK_THREADS`), удалены импорты `SelfAssessment`, `ArticleResources`, `ArticleCommentsBlock`.

### 6. Артефакты в postMessage-протоколе ✅

- `PreviewArticleData` (admin types) — добавлено поле `artifacts?: ArtifactsData`
- `toPreviewData()` — передаёт `artifacts: form.artifacts`
- `preview/page.tsx` (web) — условный рендеринг `SelfAssessment` и `ArticleResources` когда секция включена
  (`enabled: true`) и содержит непустые элементы

Цепочка данных:

```
ArtifactsTab (setValue) → react-hook-form → watch() → EditorTab
  → toPreviewData() → DevicePreview → useIframeMessaging
  → postMessage('preview:update') → web preview page
  → SelfAssessment / ArticleResources (условный рендеринг)
```
