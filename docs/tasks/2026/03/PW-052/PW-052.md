# PW-052: Метки — CRUD + API-интеграция админки

**Статус**: TODO **Создано**: 2026-03-17 **Приоритет**: Normal **Компонент**: 🐍 API + ⚙️ Admin

**Зависимости**: PW-027 (модель Tag), PW-038 (admin GET/POST эндпоинты), PW-029 (интеграция админки с API)

## Проблема

Раздел «Метки» в админке (`sections/tags/TagsSection.tsx`) работает полностью на mock-данных:

- UI поддерживает 3 режима (cloud, grid, list), поиск, группировку, статистику — но всё на `initialTags` (23 мок-тега)
- Бэкенд имеет только `GET /admin/tags` и `POST /admin/tags` — нет PUT/DELETE
- Модель `Tag` не имеет полей `color` и `group` (UI их использует)
- Схемы: `TagCreateRequest` принимает только `name` — нет `slug`, `color`, `group`
- `useTagsState` использует `useState` с моковыми данными вместо React Query
- Slug генерируется на клиенте (кириллический баг, как в PW-051), нужна серверная генерация
- Хуки `useAdminTags` / `useCreateTag` уже существуют, но не подключены к UI

## Цель

Полностью рабочий CRUD меток через API. Группировка, цвета и удаление персистентны. Мок-данные удалены.

## PW-052-A: Backend

### Миграция

Добавить в модель `Tag`:

```python
color: Mapped[str | None] = mapped_column(String(7), nullable=True)   # hex, e.g. "#ef4444"
group: Mapped[str | None] = mapped_column(String(50), nullable=True)  # "Технологии", "Дизайн", ...
```

### Схемы (`schemas/admin_tag.py`)

```python
class TagCreateRequest(BaseModel):
    name: str
    slug: str | None = None          # Авто-генерация из name если не передан
    color: str | None = None         # hex
    group: str | None = None         # Группа для фильтрации

class TagUpdateRequest(BaseModel):
    name: str | None = None
    slug: str | None = None
    color: str | None = None
    group: str | None = None

class TagAdminResponse(BaseModel):
    id: str
    name: str
    slug: str
    color: str | None = None
    group: str | None = None
    article_count: int = 0
```

### Сервисный слой (`services/tag.py` — новый файл)

Аналогично `services/category.py`:

- `get_all_tags_with_counts(db)` — JOIN-запрос (вынести из роутера)
- `create_tag(db, data)` — создание + авто-slug через `ensure_unique_tag_slug`
- `update_tag(db, id, data)` — частичное обновление, уникальность name/slug
- `delete_tag(db, id)` — удаление (проверка: если есть статьи — снять связь или запретить?)

Добавить в `services/slug.py`:

```python
def ensure_unique_tag_slug(db, slug, exclude_id=None) -> str
```

### Эндпоинты (`/admin/tags`)

| Метод  | Путь               | Описание                             | Статус    |
| ------ | ------------------ | ------------------------------------ | --------- |
| GET    | `/admin/tags`      | Список с article_count               | ✅ есть   |
| POST   | `/admin/tags`      | Создать метку                        | ✅ есть\* |
| PUT    | `/admin/tags/{id}` | Обновить метку                       | ❌ нужен  |
| DELETE | `/admin/tags/{id}` | Удалить (отвязать от статей или 409) | ❌ нужен  |

\* POST нужно расширить: принимать `slug`, `color`, `group`, использовать сервисный слой.

### Решение по удалению

При удалении метки — **отвязать от статей** (удалить записи из `article_tags`), затем удалить метку. Метки, в отличие от
категорий, не являются структурной единицей — статья может существовать без меток.

## Технические решения

- **slug**: серверная авто-генерация через `services/slug.py` (транслитерация)
- **delete**: отвязка от статей (CASCADE в `article_tags`), затем удаление — метка не структурная единица
- **color**: hex в БД, Tailwind-класс в UI (аналогично категориям)
- **group**: простая строка, nullable — фильтрация только на фронте
- **паттерн**: аналогичен PW-051 (категории) — сервисный слой, React Query мутации, toast
