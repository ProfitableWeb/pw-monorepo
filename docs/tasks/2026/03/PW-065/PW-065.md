## Постановка задачи

### Описание

Реализовать серверный AI-агент, вдохновлённый архитектурой Claude Code / OpenCode. Агент работает в цикле «думай →
действуй → наблюдай», использует инструменты для управления контентом блога (29 существующих MCP-тулов), стримит ответы
через SSE, управляет контекстным окном.

**Архитектурные слои (по аналогии с Claude Code):**

```
┌─────────────────────────────────┐
│  Context Engine                 │  ← token tracking, pruning, compaction
├─────────────────────────────────┤
│  Agent Loop                     │  ← think → act → observe cycle
├─────────────────────────────────┤
│  Tool System                    │  ← 29 MCP-инструментов как tool functions
├─────────────────────────────────┤
│  LLM Transport                  │  ← provider abstraction, streaming
├─────────────────────────────────┤
│  Provider Config (PW-064)       │  ← API keys, models, hosts из БД
└─────────────────────────────────┘
```

### Цель

Работающий AI-агент, доступный через SSE-эндпоинт. Принимает пользовательские сообщения, использует инструменты блога,
управляет контекстом сессии, стримит ответы в реальном времени.

### Критерии приемки

#### LLM Transport

- [ ] Провайдер-абстракция: единый интерфейс для OpenAI-compatible и Anthropic API
- [ ] Загрузка конфигурации из БД (PW-064 `AiProvider`)
- [ ] Стриминг токенов через SSE
- [ ] Retry с экспоненциальным backoff при ошибках провайдера

#### Agent Loop

- [ ] Цикл: получить сообщение → отправить LLM → если tool_call → выполнить → отправить результат → повторить
- [ ] Максимум итераций в одном ответе (защита от зацикливания, default: 10)
- [ ] Streaming: текст, tool calls, tool results, ошибки — всё стримится клиенту
- [ ] Системный промпт с контекстом блога и списком доступных инструментов

#### Tool System

- [ ] 29 MCP-инструментов обёрнуты как tool functions для LLM
- [ ] Каждый инструмент: name, description, parameters (JSON Schema), execute function
- [ ] Результаты инструментов возвращаются в контекст LLM
- [ ] Обработка ошибок инструментов (graceful failure)

#### Context Engine

- [ ] Подсчёт токенов (tiktoken для OpenAI-compatible, anthropic SDK для Claude)
- [ ] Pruning: удаление старых tool results при приближении к лимиту контекста
- [ ] Compaction: самосуммаризация сессии через LLM когда контекст переполняется
- [ ] Конфигурируемый лимит контекста (из `AiProvider.max_context_tokens`)

#### Sessions

- [ ] Модель `AiSession` в БД (id, user_id, title, provider_id, system_prompt, created_at, updated_at)
- [ ] Модель `AiMessage` в БД (id, session_id, role, content, tool_calls, tool_results, token_count, created_at)
- [ ] API: create session, list sessions, get session messages, send message (SSE), delete session
- [ ] Автогенерация заголовка сессии по первому сообщению

### Зависимости

- [ ] PW-064 (провайдеры — конфигурация LLM)
- [ ] `openai` Python SDK (для OpenAI-compatible провайдеров)
- [ ] `anthropic` Python SDK (для Anthropic)

#### 3. Agent Loop

```python
# apps/api/src/services/ai/agent.py
class AgentLoop:
    async def run(self, session_id, user_message) -> AsyncIterator[SSEEvent]:
        messages = load_session_messages(session_id)
        messages.append(user_message)

        for iteration in range(MAX_ITERATIONS):
            stream = transport.stream_chat(messages, tools)
            async for event in stream:
                if event.type == "text":
                    yield SSEEvent(type="text", data=event.text)
                elif event.type == "tool_call":
                    yield SSEEvent(type="tool_call", data=event.tool_call)
                    result = await execute_tool(event.tool_call)
                    yield SSEEvent(type="tool_result", data=result)
                    messages.append(tool_result_message)
                elif event.type == "end":
                    break

            if no_more_tool_calls:
                break

        # Context management
        if token_count > threshold:
            await compact_session(session_id)
```
