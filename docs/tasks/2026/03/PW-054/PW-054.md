# PW-054: Категория по умолчанию + M2M категории статей

**Статус**: TODO **Создано**: 2026-03-17 **Приоритет**: High **Компонент**: 🐍 API + ⚙️ Admin + ⚛️ Web

## Описание

WordPress-модель категорий: системная категория по умолчанию (нельзя удалить), поддержка нескольких категорий у статьи
(основная + дополнительные).

## Фаза 1: Категория по умолчанию

### Backend

1. **Миграция**: `is_default` колонка + seed «Без категории» + partial unique index
2. **Модель**: `Category.is_default: Mapped[bool]`
3. **Сервис**: `get_default_category()`, `delete_category()` перекидывает статьи на default
4. **Схемы**: `is_default` в `CategoryAdminResponse` и `CategoryResponse`
5. **API**: обновить `_to_response`, блокировать удаление default-категории

### Frontend (admin)

6. **Типы + API-клиент**: `isDefault: boolean`
7. **UI**: скрыть «Удалить» для default, бейдж «По умолчанию»

### Frontend (web)

8. **Типы + API-клиент**: `isDefault` (минорно)

## Фаза 2: M2M категории (зависит от Фазы 1)

### Backend

1. **Миграция**: `article_categories` junction table, rename `category_id` → `primary_category_id`, populate junction
2. **Модель Article**: `primary_category_id` FK + `categories` M2M через junction
3. **Модель Category**: split relationships на `primary_articles` + `articles`
4. **Сервис articles**: все запросы через primary + junction (OR-логика для страниц категорий)
5. **Сервис category**: подсчёты через junction, delete → reassign primary на default
6. **Схемы**: `primary_category` + `additional_categories` в admin, backward-compatible `category` в public

### Frontend (admin)

7. **Типы + API-клиент**: `primaryCategory` + `additionalCategories`
8. **Mappers**: `apiToFormData` / `formDataToPayload` — multi-category
9. **CardTab**: single select → «Основная категория» + multi-select «Дополнительные»

### Frontend (web)

10. **API-клиент**: `categories: list[str]` (additive, backward-compatible)

## Критерии приёмки

### Фаза 1

- [ ] При первом запуске создаётся «Без категории» (is_default=true)
- [ ] Удаление категории перекидывает статьи на default (не блокирует)
- [ ] Системную категорию нельзя удалить (кнопка скрыта, API возвращает 400)
- [ ] Можно переименовать системную категорию

### Фаза 2

- [ ] У статьи есть основная категория (обязательная) + дополнительные (опциональные)
- [ ] Страница категории показывает статьи из primary + additional
- [ ] Breadcrumbs/SEO используют primary category
- [ ] Удаление категории: reassign primary на default, junction CASCADE
- [ ] Public API backward-compatible (`category` = primary slug)

## Риски

- Rename `category_id` → `primary_category_id` — поиск строковых ссылок по всей кодобазе
- Дедупликация статей на странице категории (DISTINCT)
- Атомарность обновления editor + API + mappers
