# PW-053: Рефакторинг — общие утилиты и устранение дублирования

**Статус**: DONE **Создано**: 2026-03-17 **Завершено**: 2026-03-18 **Приоритет**: Low **Компонент**: 🐍 API + ⚙️ Admin

## Проблема

Накопившийся технический долг, выявленный при кодревью PW-051 и PW-052. Дублирование кода между доменами (категории,
метки, статьи, медиа, пользователи).

## Подзадачи

### Backend

#### 1. `_parse_uuid` — 5 копий в admin-роутерах ✅

Вынесено в `src/api/admin/utils.py` → `parse_uuid(value, label)`. Все роутеры мигрированы.

#### 2. `_to_response` — сериализация `created_at` ✅

Схема `TagAdminResponse` уже использует `datetime | None` — Pydantic сериализует автоматически. Проблема не
воспроизводилась (исправлено при создании PW-052).

### Frontend

#### 3. Inline date formatting ✅

Создана утилита `formatDate` / `formatDateTime` в `common/format.utils.ts`. Все 12 файлов с инлайн-паттерном
`new Date(str).toLocaleDateString('ru-RU', {...})` мигрированы.

#### 4. Loading spinner ✅

Создан `<LoadingSpinner />` в `common/LoadingSpinner.tsx`. Standalone-спиннеры (в центрирующем div) мигрированы. Loader2
внутри кнопок (inline loading state) оставлены — это другой паттерн.

#### 5. Mutation+toast+dialog lifecycle — ОТКЛОНЕНО

**Анализ**: сравнение CategoriesSection и TagsSection (+ useTagsState) показало:

- Совпадает ~40%: три useState, handleEdit/handleCloseDialog, toast-коллбэки
- Различия значительны:
  - Delete preconditions: Categories проверяет `isDefault`, Tags — нет
  - Delete side-effects: Tags очищает `selectedTag`, Categories — нет
  - Domain payloads: Categories — parentId/order, Tags — group
  - Специфичные хендлеры: Categories — reorder (D&D), Tags — quickAdd
- Generic hook потребовал бы столько конфигурационных коллбэков, что стал бы сложнее оригинала

**Решение**: не создавать generic hook. Текущий паттерн документирован, при появлении 3+ новых CRUD-секций пересмотреть.
