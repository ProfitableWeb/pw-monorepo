# PW-053: Рефакторинг — общие утилиты и устранение дублирования

**Статус**: TODO **Создано**: 2026-03-17 **Приоритет**: Low **Компонент**: 🐍 API + ⚙️ Admin

## Проблема

Накопившийся технический долг, выявленный при кодревью PW-051 и PW-052. Дублирование кода между доменами (категории,
метки, статьи, медиа, пользователи).

## Подзадачи

### Backend

#### 1. `_parse_uuid` — 5 копий в admin-роутерах

Одинаковая функция в `articles.py`, `categories.py`, `tags.py`, `media.py`, `users.py` с минорными вариациями (наличие
`label`, формат сообщения).

**Решение**: вынести в `src/api/admin/utils.py` с единой сигнатурой:

```python
def parse_uuid(value: str, label: str = "id") -> uuid.UUID
```

#### 2. `_to_response` — сериализация `created_at` в роутере вместо схемы

В `tags.py` роутер вручную вызывает `.isoformat()`. Pydantic может сериализовать datetime автоматически если схема
принимает `datetime | None` вместо `str | None`.

**Решение**: изменить `TagAdminResponse.created_at` на `datetime | None`, добавить `json_encoders` или
`model_serializer`.

### Frontend

#### 3. Inline date formatting — 15+ мест

`new Date(str).toLocaleDateString('ru-RU', {...})` повторяется в TagListView, TagDetailDialog, ArticlesTable,
MediaPreviewPanel, ResearchListPage и др.

**Решение**: добавить `formatDate(dateStr: string | null, options?)` в `common/format.utils.ts` (рядом с `formatBytes`).

#### 4. Loading spinner — одинаковый блок в 10+ секциях

```tsx
<div className='flex items-center justify-center py-8'>
  <Loader2 className='size-6 animate-spin text-muted-foreground' />
</div>
```

**Решение**: компонент `<LoadingSpinner />` в `ui/` или `common/`.

#### 5. Mutation+toast+dialog lifecycle — повторяется в categories и tags

`handleSave` / `handleDelete` / `handleEdit` / `handleCloseDialog` — одинаковый паттерн с
`mutation.mutate({ onSuccess: toast+close, onError: toast })`.

**Решение**: оценить возможность generic `useCrudDialogState<TForm>` хука. Если слишком много различий — оставить как
есть (документировать паттерн).
