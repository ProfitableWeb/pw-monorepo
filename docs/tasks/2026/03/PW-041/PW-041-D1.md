# PW-041-D1: Настройки → Хранилище — UI/UX админки

**Статус**: TODO **Создано**: 2026-03-09 **Приоритет**: Normal 📋 **Компонент**: ⚙️ Admin **Зависит от**: PW-041-C

## Цель

Добавить подраздел **«Хранилище»** в страницу Настроек админки — frontend-часть (компоненты, типы, вёрстка). Данные пока
на моках. Оживление через API — в PW-041-D2.

## Описание

Администратор должен наглядно видеть состояние файлового хранилища: какой бэкенд активен (Local / S3), конфигурацию,
статистику по файлам и результаты диагностики. Сейчас эта информация доступна только через `.env` и консоль Cloud.ru.

## Состав

### 1. Категория в sidebar

В `settings.constants.ts` — новая категория `storage` (иконка `HardDrive`), позиция между «Интеграции» и «Внешний вид».

### 2. Компонент StorageSettings

Паттерн: директория `storage/` по образцу `profile/` — оркестратор + assets + хук + типы.

```
settings/storage/
├── StorageSettings.tsx          # оркестратор: Tabs с 3 табами
├── useStorageInfo.ts            # хук: состояние, загрузка, тест (пока моки)
├── storage.types.ts             # интерфейсы StorageInfo, StorageTestResult
├── assets/
│   ├── OverviewTab.tsx          # бэкенд, конфиг, статус подключения
│   ├── StatsTab.tsx             # файлы, размер, по типам
│   └── DiagnosticsTab.tsx       # тест подключения
└── index.ts
```

### 3. Таб «Обзор»

- **Бейдж бэкенда**: `☁️ S3 Cloud.ru` (зелёный) или `💾 Локальный диск` (синий)
- **Конфигурация** (Card):
  - S3: бакет, регион, API endpoint, публичный endpoint
  - Local: путь на диске
  - Общее: лимит загрузки изображений (МБ), лимит прочих файлов (МБ)
- **Статус подключения**: зелёный/красный dot + latency в ms

### 4. Таб «Статистика»

- **3 карточки**: всего медиафайлов / общий размер / пользователей с аватарками
- **По типам файлов**: горизонтальные бары (image, document, video, audio) с количеством и процентом
- Форматирование: байты → KB/MB/GB, числа с разделителями

### 5. Таб «Диагностика»

- **Кнопка «Тестировать подключение»** с иконкой Play
- **Результат** — карточка с 3 шагами:
  - write ✅ 45 ms
  - read ✅ 23 ms
  - delete ✅ 31 ms
- **Итого**: суммарное время + общий success/fail бейдж
- При ошибке: красная строка с текстом

### 6. Регистрация

- `settings.constants.ts` — добавить `storage` в массив
- `SettingsPage.tsx` — добавить `case 'storage'` → `<StorageSettings />`
