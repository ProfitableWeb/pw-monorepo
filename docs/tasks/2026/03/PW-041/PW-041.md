## Цель

Создать полноценную медиа-библиотеку: загрузка файлов через API, автоматическая обработка изображений (ресайзы, EXIF),
хранение метаданных в БД, подключение существующей админской галереи к реальному API. Бакет S3 на Cloud.ru готов, но
первый этап работает на LocalStorage.

## Sync vs Async

Текущий бэкенд — синхронный (SQLAlchemy + psycopg2). Новый код PW-041 пишется **синхронно** для консистентности с
существующей кодовой базой. Миграция на async (asyncpg, async SQLAlchemy, aioboto3) — отдельная будущая задача, когда
архитектура будет развиваться в сторону агентности и MCP.

## Заметки

- Паттерн интеграции повторяет PW-040: api-client → hooks → замена моков → удаление mock файла
- `STORAGE_BACKEND=local` для разработки, S3 только на проде
- Resize pipeline по аналогии с avatar.py, но 4 размера вместо одного
- EXIF: Pillow уже умеет читать EXIF, дополнительных зависимостей не нужно
- Модель MediaFile использует `UUIDMixin` (как Article, User) — не дублировать `id` вручную
