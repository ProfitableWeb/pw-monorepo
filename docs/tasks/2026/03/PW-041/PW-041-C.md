# PW-041-C: S3Storage — Cloud.ru Evolution Object Storage

**Статус**: TODO **Создано**: 2026-03-09 **Приоритет**: Normal 📋 **Компонент**: 🐍 Backend **Зависит от**: PW-041-A

## Цель

Реализовать `S3Storage` класс — вторую реализацию `StorageBackend` для работы с Cloud.ru Evolution Object Storage
(S3-совместимый API). Переключение между Local и S3 — одна переменная `STORAGE_BACKEND` в `.env`.

## 1. Зависимость

Добавить в `apps/api/pyproject.toml`:

```toml
"boto3>=1.35.0",
```

## 2. S3Storage класс

В `apps/api/src/services/storage.py`:

```python
class S3Storage(StorageBackend):
    """Cloud.ru Evolution Object Storage (S3-совместимый). boto3."""

    def __init__(self, settings):
        self.bucket = settings.s3_bucket
        self.endpoint = settings.s3_endpoint
        self.client = boto3.client(
            "s3",
            endpoint_url=settings.s3_endpoint,
            aws_access_key_id=settings.s3_access_key,
            aws_secret_access_key=settings.s3_secret_key,
            region_name=settings.s3_region,
        )

    def save(self, path: str, data: bytes) -> str:
        self.client.put_object(
            Bucket=self.bucket,
            Key=path,
            Body=data,
            ContentType=self._guess_content_type(path),
        )
        return self.url(path)

    def delete(self, path: str) -> None:
        self.client.delete_object(Bucket=self.bucket, Key=path)

    def url(self, path: str) -> str:
        return f"{self.endpoint}/{self.bucket}/{path}"

    def exists(self, path: str) -> bool:
        try:
            self.client.head_object(Bucket=self.bucket, Key=path)
            return True
        except self.client.exceptions.ClientError:
            return False
```

**Примечание**: синхронный интерфейс, консистентный с текущим `StorageBackend` и `LocalStorage`. При будущей миграции на
async — обернуть вызовы boto3 в `asyncio.to_thread()` или перейти на `aioboto3`.

## 3. Factory функция

```python
def get_storage(settings) -> StorageBackend:
    if settings.storage_backend == "s3":
        return S3Storage(settings)
    return LocalStorage(settings)
```

Заменить текущий `storage = LocalStorage(...)` на `storage = get_storage(settings)`.

## 4. Content-Type mapping

```python
def _guess_content_type(self, path: str) -> str:
    import mimetypes
    content_type, _ = mimetypes.guess_type(path)
    return content_type or "application/octet-stream"
```

## 5. Публичный доступ к файлам

Бакет `pw-media` нужно настроить как public-read (для раздачи статики):

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::pw-media/*"
    }
  ]
}
```

Или через ACL при upload: `ACL='public-read'` в `put_object()`.

Проверить, поддерживает ли Cloud.ru Evolution bucket policies — если нет, использовать ACL.

## 6. rclone sync

Настройка описана в ADR-003. При реализации:

```ini
# /etc/rclone.conf на VM
[cloudru]
type = s3
provider = Other
endpoint = s3.cloud.ru
access_key_id = <из .env>
secret_access_key = <из .env>
force_path_style = true
```

```bash
# Cron — local → S3 (когда primary = local)
*/5 * * * * rclone sync ~/profitableweb/uploads/media/ cloudru:pw-media/media/ \
    --log-file /var/log/rclone-media.log --log-level NOTICE
```

## 7. CI/CD — прокидывание S3-секретов в .env

В `.github/workflows/deploy-gitverse-only.yml`, шаг "Write API env file", добавить:

```bash
[ -n "$STORAGE_BACKEND" ]    && ENV_CONTENT="${ENV_CONTENT}STORAGE_BACKEND=${STORAGE_BACKEND}\n"
[ -n "$S3_ENDPOINT" ]        && ENV_CONTENT="${ENV_CONTENT}S3_ENDPOINT=${S3_ENDPOINT}\n"
[ -n "$S3_BUCKET" ]          && ENV_CONTENT="${ENV_CONTENT}S3_BUCKET=${S3_BUCKET}\n"
[ -n "$S3_ACCESS_KEY" ]      && ENV_CONTENT="${ENV_CONTENT}S3_ACCESS_KEY=${S3_ACCESS_KEY}\n"
[ -n "$S3_SECRET_KEY" ]      && ENV_CONTENT="${ENV_CONTENT}S3_SECRET_KEY=${S3_SECRET_KEY}\n"
[ -n "$S3_REGION" ]          && ENV_CONTENT="${ENV_CONTENT}S3_REGION=${S3_REGION}\n"
```
