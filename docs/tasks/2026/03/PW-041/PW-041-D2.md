фронтенду из PW-041-D1, заменив моки реальными данными.

## Описание

PW-041-D1 создаёт UI на моках. Эта задача оживляет его: два новых эндпоинта API + обновление api-client.ts + подключение
хука useStorageInfo к реальным данным.

## Backend

### 1. Схемы (`schemas/storage.py`)

```python
class StorageConfigInfo(BaseModel):
    max_upload_image_mb: int           # лимит изображений
    max_upload_other_mb: int           # лимит прочих
    upload_dir: str | None = None      # только local
    bucket: str | None = None          # только s3
    region: str | None = None
    endpoint: str | None = None
    public_endpoint: str | None = None

class StorageHealthInfo(BaseModel):
    connected: bool
    latency_ms: int | None = None
    error: str | None = None

class StorageStatsInfo(BaseModel):
    media_files: int
    media_size: int                    # байты
    avatars_count: int
    by_type: dict[str, int]            # {image: 85, document: 12}

class StorageInfoResponse(BaseModel):
    backend: str                       # "local" | "s3"
    config: StorageConfigInfo
    health: StorageHealthInfo
    stats: StorageStatsInfo

class StorageTestStep(BaseModel):
    name: str                          # "write", "read", "delete"
    success: bool
    latency_ms: int
    error: str | None = None

class StorageTestResponse(BaseModel):
    success: bool
    steps: list[StorageTestStep]
    total_ms: int
```

**Безопасность**: access_key и secret_key НИКОГДА не включаются в ответ.

### 2. Сервис (`services/storage_info.py`)

#### `get_storage_info(db) → dict`

- `backend` из settings
- `config` — безопасные поля (лимиты, бакет, регион, эндпоинты, путь)
- `health`:
  - Local: `os.path.isdir()` + `os.access(W_OK)`
  - S3: `head_bucket()` с замером через `time.monotonic()`
- `stats` из БД:
  - `SELECT COUNT(*), COALESCE(SUM(size), 0) FROM media_files`
  - `SELECT COUNT(*) FROM users WHERE avatar IS NOT NULL`
  - `GROUP BY file_type`

#### `test_storage() → dict`

- Генерирует тестовый ключ `_test/{uuid}.txt`
- 3 шага с замером: write → read (exists) → delete
- Каждый шаг в try/except — не останавливает следующие

### 3. Роутер (`api/admin/storage.py`)

```
GET  /api/admin/storage/info   → ApiResponse[StorageInfoResponse]
POST /api/admin/storage/test   → ApiResponse[StorageTestResponse]
```

Оба: `Depends(get_current_admin)`.

Регистрация в `api/admin/router.py`.
