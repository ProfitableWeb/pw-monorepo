useDeleteMedia()           // DELETE /admin/media/{id} → invalidate lists + stats

## 4. Интеграция MediaSection

### 4.1. Замена состояния

| Текущее (mock)                                    | Целевое (API)                                   |
| ------------------------------------------------- | ----------------------------------------------- |
| `useState<MediaFile[]>(initialFiles)`             | `useMediaList({ page, type, purpose, search })` |
| Локальный фильтр по name/type/purpose             | Query params на сервер                          |
| `setFiles(files.filter(...))` при удалении        | `useDeleteMedia()` → invalidation               |
| `setIsUploading(true)` + setTimeout               | `useUploadMedia()` с progress                   |
| `setPreviewFile(file)` → локальное редактирование | `useUpdateMedia()` при сохранении SEO/EXIF      |

### 4.2. Upload flow

```
1. Пользователь drag-and-drop или file picker
2. useUploadMedia.mutate(file)
3. Progress tracking (XMLHttpRequest onprogress или fetch с ReadableStream)
4. onSuccess: invalidate lists + stats, toast "Загружено"
5. onError: toast с ошибкой
```

### 4.3. Preview/Edit flow

```
1. Клик по файлу → MediaPreviewDialog открывается
2. useMedia(fileId) — загрузка полных данных (EXIF, resizes, usedIn)
3. Редактирование slug/alt/caption/purposes в форме
4. Сохранение → useUpdateMedia.mutate({ slug, alt, caption, purposes })
5. onSuccess: invalidate detail, toast "Сохранено"
```

### 4.4. Delete flow

```
1. Кнопка удаления → confirm dialog
2. useDeleteMedia.mutate(fileId)
3. onSuccess: invalidate lists + stats, закрыть preview если открыт
```

### 4.5. Stats (sidebar)

Заменить `mockFiles.length` / `mockFiles.reduce(size)` на `useMediaStats()` для:

- Общее количество файлов
- Общий объём
- Количество по типам
- Список purposes для фильтра
