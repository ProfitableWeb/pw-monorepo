# PW-043: Docker-контейнеризация и два контура деплоя (dev/prod)

**Статус**: DONE **Создано**: 2026-03-11 **Завершено**: 2026-03-15 **Приоритет**: High **Компонент**: DevOps

## Цель

Перевести деплой с голого PM2 на Docker Compose. Два контура (dev + prod) на одной VM. GitHub как primary Git, Gitea на
VM как зеркало и запасной аэродром.

## Контекст

### Текущее состояние

| Компонент       | Состояние                                                  |
| --------------- | ---------------------------------------------------------- |
| Процессы        | PM2 (api, web, admin)                                      |
| Сборка          | `git pull` → `bun install` → `turbo build` → `pm2 restart` |
| Контуры         | Один (prod)                                                |
| Контейнеризация | Нет Docker                                                 |
| Git-хостинг     | GitVerse (primary) + GitHub (mirror)                       |
| Изоляция        | Нет — все процессы в одном окружении                       |
| Откат           | Ручной `git revert` + пересборка                           |
| Мониторинг VM   | Cloud.ru стандартные метрики (CPU, сеть)                   |

### VM: Cloud.ru

- **Флейвор**: free-tier-2-4 (2 vCPU, 4 ГБ RAM)
- **Гарантированная доля CPU**: 10% (burst-модель)
- **ОС**: Ubuntu 22.04
- **S3**: pw-media (15 ГБ, Object Storage)
- **IP**: 213.171.25.187

### Проблемы

1. **GitVerse нестабилен** — 3+ дня ошибок, CI/CD зависит от clone оттуда
2. **Нет изоляции** — dev-тестирование невозможно без риска для prod
3. **Откат медленный** — нет готовых образов, каждый раз пересборка
4. **Нет dev-контура** — всё проверяется только на проде или локально

## Архитектура

### Целевая схема

```
VM (Cloud.ru, 4 ГБ RAM)
├── nginx (host-level, reverse proxy)
│   ├── profitableweb.ru       → prod контур (:3000, :8000, :3001)
│   ├── dev.profitableweb.ru   → dev контур  (:3100, :8100, :3101)
│   └── git.profitableweb.ru   → Gitea       (:3300)
│
├── docker-compose.infra.yml
│   ├── gitea          :3300
│   └── gitea-postgres :5434
│
├── docker-compose.prod.yml
│   ├── api            :8000
│   ├── web            :3000
│   ├── admin          :3001
│   └── postgres       :5432
│
└── docker-compose.dev.yml
    ├── api            :8100
    ├── web            :3100
    ├── admin          :3101
    └── postgres       :5433 (отдельная БД)
```

### Git flow

```
feature/* → develop → master
               ↓          ↓
          dev контур   prod контур
```

### Бюджет RAM (оценка)

| Компонент                           | RAM         |
| ----------------------------------- | ----------- |
| ОС + nginx (host)                   | ~300 МБ     |
| Prod (api + web + admin + postgres) | ~500 МБ     |
| Dev (api + web + admin + postgres)  | ~500 МБ     |
| Gitea + postgres                    | ~200 МБ     |
| Docker daemon                       | ~200 МБ     |
| **Итого**                           | **~1.7 ГБ** |

Остаётся ~2.3 ГБ на кэши и пики. При необходимости dev-контур можно останавливать.

### Принципиальные решения

| Решение            | Выбор               | Почему                                        |
| ------------------ | ------------------- | --------------------------------------------- |
| Container registry | Нет, build на VM    | Одна VM, один разработчик, лишний трафик      |
| Kubernetes         | Нет, Docker Compose | 3 сервиса на 1 ноде, K8s — overkill           |
| Git primary        | GitHub              | Стабильный, Actions бесплатны                 |
| Git mirror         | Gitea на VM         | Полный контроль, автономность                 |
| Turbo              | Остаётся            | Build orchestration внутри Docker build stage |
| PM2                | Уходит              | Docker restart policy заменяет                |

## PW-043-A: Dockerfiles

### api (apps/api)

```dockerfile
FROM python:3.12-slim
# uv install, copy pyproject.toml, uv sync
# copy src/, alembic/
# CMD: uv run alembic upgrade head && uv run uvicorn src.main:app
```

### web (apps/web)

```dockerfile
# Stage 1: build (bun + turbo)
FROM oven/bun:1 AS builder
# bun install, turbo build --filter=@profitable-web/web

# Stage 2: run
FROM oven/bun:1-slim
# copy .next/standalone, .next/static, public
# CMD: bun run server.js
```

### admin (apps/admin)

```dockerfile
# Stage 1: build
FROM oven/bun:1 AS builder
# bun install, turbo build --filter=@profitable-web/admin

# Stage 2: serve
FROM nginx:alpine
# copy dist/ to /usr/share/nginx/html
```

**Ключевые моменты:**

- Multi-stage builds для минимального размера образов
- `.dockerignore` для исключения node_modules, .git
- Turbo prune для изоляции зависимостей каждого app
- Admin — статика через nginx (не нужен Bun runtime в проде)

## PW-043-B: docker-compose.prod.yml

```yaml
services:
  postgres:
    image: postgres:16-alpine
    volumes: [pw-prod-data:/var/lib/postgresql/data]
    environment: [POSTGRES_DB, POSTGRES_USER, POSTGRES_PASSWORD]
    restart: unless-stopped

  api:
    build: { context: ., dockerfile: apps/api/Dockerfile }
    depends_on: [postgres]
    env_file: .env.prod
    ports: ['127.0.0.1:8000:8000']
    restart: unless-stopped

  web:
    build: { context: ., dockerfile: apps/web/Dockerfile }
    depends_on: [api]
    ports: ['127.0.0.1:3000:3000']
    restart: unless-stopped

  admin:
    build: { context: ., dockerfile: apps/admin/Dockerfile }
    ports: ['127.0.0.1:3001:80']
    restart: unless-stopped

volumes:
  pw-prod-data:
```

## PW-043-E: CI/CD (GitHub Actions)

### Prod (push to master)

```yaml
on:
  push:
    branches: [master]
steps:
  - SSH to VM
  - git pull origin master
  - docker compose -f docker-compose.prod.yml up -d --build
  - health check
```

### Dev (push to develop)

```yaml
on:
  push:
    branches: [develop]
steps:
  - SSH to VM
  - git pull origin develop
  - docker compose -f docker-compose.dev.yml up -d --build
  - health check dev.profitableweb.ru
```

## Что НЕ делаем (осознанно)

| Что                  | Почему нет                                |
| -------------------- | ----------------------------------------- |
| Kubernetes           | 3 сервиса на 1 VM, overkill               |
| Container registry   | Build на VM, нет смысла в push/pull       |
| Docker Swarm         | Одна нода, не нужна оркестрация           |
| Traefik вместо nginx | nginx уже настроен, работает, знаком      |
| SSL через Docker     | Let's Encrypt на host nginx, проще        |
| Watchtower           | Автообновление образов опасно для prod    |
| Prometheus + Grafana | 2-4 ГБ RAM на мониторинг > всё приложение |

## Риски

| Риск                             | Митигация                                                              |
| -------------------------------- | ---------------------------------------------------------------------- |
| Docker build медленный (10% CPU) | Кэширование слоёв, multi-stage, build в off-peak                       |
| 4 ГБ RAM не хватит               | Dev-контур можно останавливать: `docker compose -f dev.yml stop`       |
| Миграция сломает prod            | Сначала поднять Docker рядом с PM2, переключить nginx, потом убить PM2 |
| Gitea жрёт ресурсы               | Мониторить, при необходимости — SQLite вместо PostgreSQL               |
