## Постановка задачи

### Описание

Декларативно проектная документация (`.claude/CLAUDE.md`, `apps/admin/CLAUDE.md`, `docs/architecture/`, `README.md`)
указывает Litestar как backend-фреймворк. Фактически весь код `apps/api` использует FastAPI: 28 импортов `from fastapi`,
0 импортов `litestar`, включая `main.py`, все роутеры, зависимости авторизации, MCP-интеграцию.

Задача — устранить рассогласование, выполнив миграцию кода на Litestar.

### Цель

Реальная архитектура соответствует декларации в CLAUDE.md и архитектурной документации. Litestar используется как
основной web-фреймворк бэкенда.

### Критерии приемки

- [ ] `apps/api/src/main.py` использует Litestar app + Litestar lifespan
- [ ] Все роутеры (`api/`, `api/admin/`) переписаны на `litestar.Router` / `litestar.Controller`
- [ ] Зависимости (`auth/dependencies.py`, `core/database.py`) портированы на Litestar DI
- [ ] CORS, request middleware, structured logging работают
- [ ] MCP-сервер монтируется как ASGI-app с корректным lifespan (Litestar поддерживает mount по аналогии с FastAPI)
- [ ] OpenAPI-спецификация генерируется (Litestar OpenAPIController)
- [ ] Все существующие тесты проходят
- [ ] `pyproject.toml`: убран `fastapi`, добавлен `litestar`
- [ ] PM2 ecosystem (`uvicorn`) совместим — Litestar работает с uvicorn

### Зависимости

- [ ] `litestar` (~2.x)
- [ ] Возможно: `litestar[standard]` для встроенных pydantic-интеграций

### Стратегия миграции

Поэтапно по слоям, с возможностью отката на каждом этапе:

1. **Подготовка**: добавить `litestar` в зависимости рядом с FastAPI, не удаляя FastAPI
2. **Параллельный запуск**: создать минимальный Litestar-app на отдельном порту (8001) для smoke-тестов
3. **Миграция роутеров**: по одному из `api/` (auth → users → articles → ...) с тестами
4. **Миграция middleware**: CORS, RequestLoggingMiddleware, CorrelationId
5. **Миграция auth dependencies**: `get_current_user/admin/optional_user` под Litestar DI
6. **Миграция MCP-mount**: проверить совместимость FastMCP с Litestar
7. **Финал**: переключить `main.py` полностью на Litestar, убрать FastAPI из зависимостей

### Риски

- **MCP-сервер** (`src/mcp/`) — FastMCP создаёт ASGI-app, но lifespan интеграция с Litestar требует проверки
- **Pydantic-схемы** — оба фреймворка поддерживают Pydantic v2, но валидация ошибок может отличаться по формату
- **OpenAPI-документация** — структура отличается, существующие клиенты могут потребовать перегенерации
- **Time investment** — оценка 16-24 часа сосредоточенной работы

### Альтернатива

Если миграция признаётся нецелесообразной — обновить документацию обратно на FastAPI (CLAUDE.md и пр.). Это решение
принять до начала миграции.
