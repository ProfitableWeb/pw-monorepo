# PW-061: MCP-сервер для управления блогом

**Статус:** TODO **Приоритет:** Высокий **Дата:** 2026-03-21

## Суть

MCP (Model Context Protocol) сервер, позволяющий любому AI-клиенту (Claude Desktop, Cursor, Claude Code, Claude.ai)
управлять блогом ProfitableWeb — создавать статьи, управлять категориями, загружать медиа, анализировать SEO.

## Зачем

- Управление блогом из любого AI-ассистента без открытия админки
- Создание и редактирование контента прямо в процессе работы с AI
- Автоматизация рутины: SEO-анализ, генерация slug, публикация по расписанию
- Единый интерфейс для Claude Desktop, Cursor, VS Code, Claude.ai

## Архитектурные решения

### Где живёт MCP-сервер

**Внутри `apps/api/src/mcp/`** — модуль в существующем FastAPI backend.

Плюсы:

- Переиспользует service layer напрямую (те же функции что и admin API)
- Общая инфраструктура: БД, auth, config, модели
- Нет HTTP-оверхеда (сервисы вызываются как Python-функции)

Два режима работы:

1. **Integrated** — ASGI sub-app на FastAPI по пути `/mcp` (для production/remote)
2. **Standalone** — `uv run python -m src.mcp` через stdio (для Claude Desktop/Cursor)

### Транспорт

| Транспорт           | Применение                               | Приоритет    |
| ------------------- | ---------------------------------------- | ------------ |
| **Streamable HTTP** | Remote: Claude.ai connectors, CI/CD, API | Основной     |
| **stdio**           | Local: Claude Desktop, Cursor, VS Code   | Вторичный    |
| SSE                 | Legacy (deprecated в MCP spec)           | Не реализуем |

### SDK

**Официальный `mcp` Python SDK (v1.x)** с `FastMCP`.

Не `fastapi-mcp` (auto-convert) — нужны curated tools с осмысленными именами и описаниями, а не механическая конвертация
REST endpoints.

### Аутентификация

**API-ключи** (не JWT, не OAuth 2.1).

- JWT — для браузерных сессий, не подходит для MCP-клиентов
- OAuth 2.1 — для мультитенантных SaaS, overkill для single-tenant блога
- API-ключи — простые, stateless, с гранулярными scope

## Модель безопасности

### Таблица `mcp_api_keys`

```
- id: UUID (PK)
- name: str              — человекочитаемое имя ("Claude Desktop — Николай")
- key_hash: str           — bcrypt hash ключа
- key_prefix: str         — первые 8 символов для отображения ("pw_mcp_a")
- user_id: UUID (FK)      — владелец (определяет author_id для контента)
- scope: enum             — read | write | admin
- is_active: bool
- last_used_at: datetime?
- expires_at: datetime?
- created_at: datetime
```

Формат ключа: `pw_mcp_{40_hex_chars}`. Показывается один раз при создании.

### Scope

| Scope   | Операции                                                             |
| ------- | -------------------------------------------------------------------- |
| `read`  | `list_*`, `get_*`, `search_*`, `analyze_*`                           |
| `write` | read + `create_*`, `update_*`, `publish_*`, `schedule_*`, `upload_*` |
| `admin` | write + `delete_*`, `update_seo_settings`, `get_audit_log`           |

### Rate limiting

Redis sliding window по API-ключу:

- `read`: 60 req/min
- `write`: 30 req/min
- `admin`: 10 req/min

### Аудит

Все MCP-вызовы пишутся в `audit_logs`:

- `action`: `mcp.{tool_name}`
- `user_id`: из API-ключа
- `changes`: параметры вызова (без тел больших статей)

## Admin UI: страница MCP

### Расположение

Система → MCP (новый пункт в sidebar секции "Система")

### Структура страницы

Два основных блока:

#### Блок 1: Ключи доступа

Таблица API-ключей с колонками:

- Имя, Префикс (`pw_mcp_a8f3...`), Scope, Статус, Последнее использование, Создан
- Действия: деактивировать, удалить

Кнопка **"Создать ключ"** — диалог:

- Имя ключа (напр. "Claude Desktop — Николай")
- Права: `read` / `write` / `admin`
- Срок действия: без ограничений / 30 дн / 90 дн / свой

После создания — модалка **"Ключ создан"**:

- Ключ `pw_mcp_a8f3...` с кнопкой копирования
- Предупреждение: "Ключ показывается один раз. Сохраните его сейчас."
- Ниже — **готовые конфиги с подставленным ключом** (табы по клиентам)

#### Блок 2: Инструкция по подключению

Постоянный блок с табами по AI-клиентам:

**Tab: Claude Desktop**

```
1. Откройте Claude Desktop → Settings → Developer → Edit Config
2. Вставьте конфигурацию:

{
  "mcpServers": {
    "profitableweb": {
      "url": "https://api.profitableweb.ru/mcp",
      "headers": {
        "Authorization": "Bearer <ваш_ключ>"
      }
    }
  }
}

3. Перезапустите Claude Desktop
4. В новом чате появятся инструменты ProfitableWeb (29 tools)
```

**Tab: Cursor**

```
1. Создайте файл .cursor/mcp.json в корне проекта
2. Вставьте:

{
  "mcpServers": {
    "profitableweb": {
      "url": "https://api.profitableweb.ru/mcp",
      "headers": {
        "Authorization": "Bearer <ваш_ключ>"
      }
    }
  }
}

3. Перезапустите Cursor
```

**Tab: Claude Code**

```
1. Откройте терминал и выполните:

claude mcp add profitableweb \
  --transport http \
  --url https://api.profitableweb.ru/mcp \
  --header "Authorization: Bearer <ваш_ключ>"

2. Или добавьте в .claude/settings.local.json:

{
  "mcpServers": {
    "profitableweb": {
      "command": "uv",
      "args": ["run", "--directory", "/path/to/apps/api", "python", "-m", "src.mcp"],
      "env": {
        "PW_MCP_API_KEY": "<ваш_ключ>"
      }
    }
  }
}
```

**Tab: API (для разработчиков)**

```bash
# Список доступных tools
curl -X POST https://api.profitableweb.ru/mcp \
  -H "Authorization: Bearer pw_mcp_..." \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"tools/list","id":1}'

# Список статей
curl -X POST https://api.profitableweb.ru/mcp \
  -H "Authorization: Bearer pw_mcp_..." \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"tools/call","params":{"name":"list_articles","arguments":{"limit":5}},"id":2}'
```

## Фазы реализации

### PW-061-A: Mock-first Admin UI

Цель: полностью рабочая страница MCP в админке на mock-данных. Можно ревьюить UI/UX до написания backend.

1. Навигация: пункт "MCP" в sidebar (секция "Система")

3. `McpApiKeys.tsx` — таблица ключей (mock-данные: 2-3 ключа)
4. Диалог создания ключа (имя, scope, срок действия)
5. Модалка "Ключ создан" с копированием + конфиги по табам
6. `McpConnectionGuide.tsx` — инструкция по подключению (табы: Claude Desktop, Cursor, Claude Code, API)
7. `McpConnectionTest.tsx` — виджет "Проверить подключение" (mock: всегда ✅)

**Результат:** можно открыть в браузере, пощупать весь флоу, дать фидбек.

### PW-061-B: Backend (MCP server + API keys)

Цель: работающий MCP-сервер, подключаемый из Claude Desktop.

1. Модель `McpApiKey` + миграция Alembic
2. Сервис `mcp_key.py` — create, list, revoke, verify
3. Admin API: `POST/GET/DELETE /api/admin/mcp-keys`
4. MCP server skeleton: `src/mcp/server.py` (`FastMCP`)
5. Auth: `src/mcp/auth.py` — верификация API-ключа, scope
6. Tools: articles (9), categories (4), tags (4), media (4), SEO (3), system (3), content (2)
7. Scope enforcement (декоратор)
8. Audit logging (`mcp.*` actions)
9. Mount на FastAPI: `/mcp` (Streamable HTTP)
10. stdio entry point: `src/mcp/__main__.py`
11. MCP resources (editorial guide, categories)

**Результат:** `claude mcp add profitableweb` — и всё работает.

### PW-061-C: Интеграция front + back

Цель: заменить mock-данные на реальный API, всё работает end-to-end.

1. API-клиент: функции для MCP ключей в `api-client.ts`
2. `McpApiKeys.tsx` → реальный CRUD через API
3. Диалог создания → реальный `POST /api/admin/mcp-keys`, показ ключа
4. `McpConnectionTest.tsx` → реальный вызов `tools/list` на `/mcp`
5. `McpAuditLog.tsx` → реальный аудит-лог с фильтром `mcp.*`
6. Rate limiting (Redis)
7. Деплой, проверка на проде

**Результат:** полный production-ready MCP с управлением из админки.

## Риски

| Риск                           | Митигация                                                |
| ------------------------------ | -------------------------------------------------------- |
| Большие статьи в ответах tools | Truncate content в list, полный — только в `get_article` |
| Base64 upload > 10MB           | Лимит, для больших файлов — админка                      |
| Утечка API-ключа               | Показ раз, hash storage, revocation, аудит               |
| MCP SDK breaking changes       | Pin v1.x                                                 |
