## Постановка задачи

### Описание

Заменить мок-реализацию AI-чата в админке на реальное взаимодействие с серверным AI-агентом (PW-065). Реализовать
полноценный чат-интерфейс с стримингом ответов, визуализацией tool calls, управлением сессиями и отображением
токеномики.

Текущее состояние: AI Center содержит мок-чат с фейковыми сессиями и 2-секундной задержкой (`ai-store.ts`). Нужно
подключить к реальному SSE-бэкенду.

### Цель

Пользователь (админ) открывает AI-чат, пишет запрос, видит стриминг ответа в реальном времени, наблюдает вызовы
инструментов (с разворачиваемыми деталями), может управлять сессиями.

### Критерии приемки

#### Чат-интерфейс

- [ ] Стриминг текста (посимвольное появление ответа через SSE)
- [ ] Markdown-рендеринг ответов
- [ ] Визуализация tool calls: иконка инструмента, имя, параметры (сворачиваемо), результат (сворачиваемо), статус
      (pending/success/error)
- [ ] Индикатор «думает...» во время tool execution
- [ ] Автоскролл к новым сообщениям
- [ ] Кнопка отмены генерации (abort SSE connection)

#### Сессии

- [ ] Список сессий в сайдбаре (заголовок, дата, превью)
- [ ] Создание новой сессии
- [ ] Переключение между сессиями (загрузка истории)
- [ ] Удаление сессии
- [ ] Автогенерация заголовка после первого сообщения

#### Провайдер

- [ ] Отображение текущего активного провайдера (имя, модель)
- [ ] Возможность переключить провайдер для сессии (select)

#### Токеномика

- [ ] Отображение использования токенов в сессии (input/output/total)
- [ ] Прогресс-бар заполнения контекстного окна
- [ ] Уведомление при compaction (суммаризация контекста)

#### UX

- [ ] Горячие клавиши: Enter → отправить, Shift+Enter → новая строка
- [ ] Пустой state: подсказки что можно спросить (примеры запросов)
- [ ] Responsive: работает на разных ширинах панели

### Зависимости

### Техническое решение

#### 1. Обновление ai-store

Заменить мок-логику в `ai-store.ts` на реальные API-вызовы:

```typescript
// Сессии: GET /api/admin/ai/sessions
// Сообщения: GET /api/admin/ai/sessions/:id/messages
// Отправка: POST /api/admin/ai/sessions/:id/messages (SSE)
// Удаление: DELETE /api/admin/ai/sessions/:id
// Провайдеры: GET /api/admin/ai-providers
```

#### 2. SSE-клиент

```typescript
// lib/sse-client.ts

    method: 'POST',
    body: JSON.stringify({ message }),
    headers: { 'Content-Type': 'application/json' },
  });
  const reader = response.body.getReader();
  // parse SSE events...
}
```

#### 3. Формат SSE-событий

```
event: text
data: {"delta": "Вот список "}

event: text
data: {"delta": "последних статей:"}
```
