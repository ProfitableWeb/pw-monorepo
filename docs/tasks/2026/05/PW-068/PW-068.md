## Постановка задачи

### Описание

В проекте накапливаются правила и документация для работы AI-кодинг-агентов: Cursor rules, Claude context, шаблоны
секционных README, memory-файл `AGENTS.md`, подход к context headers в коде. Нужно оформить это как явную
агентно-ориентированную практику проекта, чтобы человек и агент одинаково понимали, где лежит контекст и как его
поддерживать.

### Цель

Создать устойчивую систему агентного контекста: короткие специализированные правила, документацию по agentic
development, шаблоны для frontend/backend context headers и гигиену обновления памяти без локальных/секретных данных.

### Критерии приемки

- [ ] `.cursor/rules/` содержит короткие специализированные правила по областям: git, docs, backend, frontend, state,
      theming, TypeScript, LLM prompts, context headers
- [ ] `docs/agentic-development.md` описывает карту агентного контекста и правила context headers
- [ ] `docs/templates/` структурирован по `frontend/`, `backend/`, `shared/`
- [ ] Шаблоны покрывают frontend JSDoc и Python module docstring
- [x] `AGENTS.md` содержит только high-signal recurring preferences и durable workspace facts
- [x] Локальные индексы continual-learning не попадают в репозиторий или явно игнорируются
- [ ] В документации нет противоречий по активному деплой-пути и источникам истины

### Зависимости

- Нет внешних зависимостей.

### Техническое решение

- Держать Cursor rules маленькими: один `.mdc` — одна тема.
- Разделить шаблоны контекста по языкам и назначению.
- Использовать `docs/agentic-development.md` как обзорный документ по агентной разработке.
- Хранить durable agent memory в `AGENTS.md`, но не коммитить локальные transcript index-файлы с абсолютными путями.
