## Постановка задачи

### Описание

Сейчас ProfitableWeb использует несколько слоёв агентного контекста: `AGENTS.md`, `.claude/CLAUDE.md`,
`apps/*/CLAUDE.md`, `.cursor/rules/*.mdc`, `docs/agentic-development.md` и `docs/templates/`. Это работает, но при
подключении новых инструментов (Codex, Copilot, OpenCode, Gemini, Windsurf) может появиться рассинхронизация правил.

Нужно отдельно исследовать и спроектировать переносимую схему правил: оставить текущую модель или ввести canonical
source (`.agents/` или `docs/ai-rules-base.md`) и генерацию tool-specific adapters.

### Цель

Сформировать архитектуру multi-agent portability для ProfitableWeb без дублирования правил и без потери преимуществ
tool-specific форматов вроде `.cursor/rules/*.mdc`.

### Критерии приемки

- [x] Зафиксировано, какие инструменты должны поддерживаться на первом этапе (Cursor, Claude Code, Codex, Copilot,
      OpenCode и совместимые AGENTS.md-инструменты) — матрица в `ADR-004` и `docs/agentic-development.md`
- [x] Описана роль `AGENTS.md`: portable Tier 0 в root; nested-AGENTS.md не вводятся, app-specific остаётся в
      `apps/*/CLAUDE.md`
- [x] Принято решение: оставить shared (`AGENTS.md`) + tool-specific (`.cursor/rules/`, `.claude/CLAUDE.md`,
      `.github/copilot-instructions.md`); canonical `.agents/` не вводится — обоснование в ADR-004
- [x] Sync-скрипт явно отложен; зафиксированы триггеры для пересмотра (3+ инструмента + регулярные расхождения)
- [x] Описано, что генерируется автоматически (ничего на этом этапе) и что остаётся tool-specific (`.mdc`-фронтматтер
      Cursor, nested-CLAUDE.md Claude Code) — таблица в ADR-004
- [x] MCP описан как отдельный runtime-слой памяти и инструментов, не замена rules — раздел в ADR-004
- [x] `docs/agentic-development.md` обновлён: новая «Карта агентного контекста» с tier-таблицей, принципами
      переносимости и матрицей инструментов

### Зависимости

- PW-068 — агентно-ориентированная разработка и правила контекста.

### Техническое решение

Предварительная гипотеза:

- `AGENTS.md` — portable high-signal слой.
- `.cursor/rules/*.mdc` — granular Cursor layer с `globs`.
- `.claude/CLAUDE.md` — Claude Code entrypoint со ссылками на `AGENTS.md` и `.cursor/rules`.
- `.agents/` вводить только если появится sync-скрипт и реальная потребность поддерживать 3+ agent tools.
