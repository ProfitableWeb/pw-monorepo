# ADR-001: Инфраструктура базы данных

> **Статус**: Принято **Дата**: 2026-02-15 **Связанные задачи**: PW-027, PW-030, PW-031

## Контекст

Backend API (PW-027) использует PostgreSQL + SQLAlchemy. Тестовый контур на Cloud.ru (VM 213.171.25.187) до сих пор
работал без БД — API отдавал данные из старой mock-версии. С внедрением аутентификации (PW-030) и реального CRUD API
требуется полноценная БД на тестовом контуре.

## Решения

### 1. PostgreSQL устанавливается на VM (не managed)

**Выбрано**: PostgreSQL на той же VM через `apt install`.

**Причины**:

- Тестовый контур, не продакшн — не нужны managed-бэкапы и SLA
- Бесплатно, минимальная задержка (localhost)
- Достаточно для текущей нагрузки (1 сервер, мало пользователей)

**Альтернатива**: Managed PostgreSQL Cloud.ru — отложено до продакшн-контура.

### 2. Схема БД синхронизируется через Alembic

**Механизм**: `uv run alembic upgrade head` запускается автоматически при каждом деплое (в CI/CD workflow).

**Гарантии**:

- Миграции идемпотентны (Alembic отслеживает версию через `alembic_version`)
- При ошибке миграции деплой останавливается (`set -e`)

### 3. Начальные данные загружаются seed-скриптом

**Механизм**: `uv run python scripts/seed.py` — идемпотентный скрипт.

**Принцип**: `INSERT ... ON CONFLICT DO NOTHING` — безопасен для повторного запуска.

**Что загружает seed**:

- Категории (справочник)
- Тестовые статьи (для проверки фронтенда)
- Не загружает пользователей (создаются через OAuth)

**Запуск**: вручную после первого деплоя, далее при необходимости.

### 4. Переменные окружения передаются через CI/CD secrets

**Механизм**: GitVerse secrets → SSH → запись в `apps/api/.env` на VM.

**Переменные**: DATABASE_URL, JWT_SECRET, OAuth credentials, CORS_ORIGINS, frontend/admin URLs.

### 5. Первый администратор назначается вручную

**Порядок**:

1. Пользователь входит через OAuth → создаётся с role=viewer
2. На VM: `uv run python scripts/promote_admin.py user@email.com`

---

## Схема синхронизации

```
Локальная разработка          Тестовый контур (VM)
──────────────────           ───────────────────
PostgreSQL (localhost)        PostgreSQL (localhost)
    │                             │
    ├── Схема: Alembic ──push──> Alembic upgrade head (CI/CD)
    ├── Справочники: seed.py ──> seed.py (вручную)
    ├── Контент: ручной ввод     Через админку
    └── .env: локальный          Из GitVerse secrets (CI/CD)
```

## Последствия

- При добавлении новой таблицы/колонки: создать Alembic миграцию → деплой применит автоматически
- При добавлении новой категории: добавить в seed.py → запустить на VM вручную (или через админку)
- При переходе на продакшн: заменить PostgreSQL на VM на managed сервис, изменить DATABASE_URL в secrets
