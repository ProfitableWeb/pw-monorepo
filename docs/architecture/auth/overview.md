# Аутентификация — обзор

## JWT Flow

```
Пользователь          Frontend            API
     │                    │                 │
     │── OAuth login ────▶│                 │
     │                    │── /auth/login ─▶│
     │                    │                 │── проверка OAuth
     │                    │◀── Set-Cookie ──│   access_token (15min)
     │                    │    (httpOnly)   │   refresh_token (7d)
     │                    │                 │
     │── действие ───────▶│                 │
     │                    │── /api/... ────▶│── проверка access_token
     │                    │◀── данные ──────│
     │                    │                 │
     │   (через 15 мин)   │                 │
     │── действие ───────▶│                 │
     │                    │── /api/... ────▶│── 401 (токен истёк)
     │                    │── /auth/refresh▶│── проверка refresh_token
     │                    │◀── Set-Cookie ──│   новый access + refresh
     │                    │── /api/... ────▶│── OK
     │                    │◀── данные ──────│
```

## Токены

| Параметр    | Access Token       | Refresh Token          |
| ----------- | ------------------ | ---------------------- |
| Время жизни | 15 минут           | 7 дней                 |
| Хранение    | httpOnly cookie    | httpOnly cookie        |
| SameSite    | Lax                | Lax                    |
| Secure      | true (prod)        | true (prod)            |
| Содержимое  | user_id, role, exp | user_id, token_id, exp |

### Ротация refresh-токена

При обновлении access-токена refresh-токен **также заменяется** (rotation). Старый refresh-токен становится невалидным.
Это предотвращает повторное использование украденного токена.

## Роли

| Роль     | Блог (web)      | Админ-панель (admin)                     |
| -------- | --------------- | ---------------------------------------- |
| `admin`  | Полный доступ   | Полный доступ, управление пользователями |
| `editor` | Полный доступ   | Редактирование всех статей               |
| `author` | Комментирование | Создание и редактирование своих статей   |
| `viewer` | Комментирование | Нет доступа (редирект)                   |
| Гость    | Только чтение   | Нет доступа                              |

Новые OAuth-пользователи получают роль `viewer` по умолчанию.

## Защита админ-панели

Middleware в `apps/admin` проверяет:

1. Наличие access_token в cookies
2. Роль пользователя: `admin` или `editor`
3. При отсутствии/невалидности — редирект на `/login`

## Эндпоинты

| Метод | Путь                            | Описание                    |
| ----- | ------------------------------- | --------------------------- |
| GET   | `/api/auth/{provider}/login`    | Редирект на OAuth-провайдер |
| GET   | `/api/auth/{provider}/callback` | Обработка OAuth callback    |
| POST  | `/api/auth/refresh`             | Обновление токенов          |
| POST  | `/api/auth/logout`              | Удаление cookies            |
| GET   | `/api/auth/me`                  | Текущий пользователь        |
