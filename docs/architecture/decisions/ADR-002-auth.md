# ADR-002: Аутентификация

**Дата**: 2026-02-12

## Статус

Принято

## Контекст

Нужна система аутентификации для админ-панели и комментариев на блоге. Требования: OAuth через популярные российские и
международные провайдеры, ролевой доступ, безопасное хранение токенов.

## Решение

JWT в httpOnly cookies с OAuth через Yandex, Google и Telegram.

### Детали реализации

**Токены**:

- `access_token`: 15 минут, httpOnly cookie
- `refresh_token`: 7 дней, httpOnly cookie
- Автоматическая ротация refresh-токена при обновлении

**Почему cookies, а не localStorage**:

- Защита от XSS-атак (JS не может прочитать httpOnly cookie)
- Автоматическая отправка с каждым запросом (не нужен interceptor)
- CSRF защищён через SameSite=Lax

**OAuth-провайдеры**: Yandex, Google, Telegram

**Роли**:

| Роль     | Доступ                                            |
| -------- | ------------------------------------------------- |
| `admin`  | Полный доступ, управление пользователями          |
| `editor` | Редактирование всех статей                        |
| `author` | Создание и редактирование своих статей            |
| `viewer` | Комментирование (OAuth-пользователи по умолчанию) |

**Защита админ-панели**: middleware проверяет роль `admin` или `editor`, иначе редирект на `/login`.

## Альтернативы

| Альтернатива       | Плюсы              | Минусы                                              | Почему отклонена    |
| ------------------ | ------------------ | --------------------------------------------------- | ------------------- |
| JWT в localStorage | Простая реализация | Уязвим к XSS                                        | Безопасность важнее |
| Session-based auth | Простота           | Stateful, проблемы с масштабированием               | JWT гибче           |
| NextAuth.js        | Готовое решение    | Привязка к Next.js, не подходит для FastAPI backend | Бэкенд на Python    |

## Последствия

### Положительные

- Безопасное хранение токенов (httpOnly)
- Единая авторизация для web и admin через один API
- Гибкие роли для будущего расширения

### Отрицательные

- Сложнее отладка (токены не видны в DevTools)
- Нужна регистрация приложения у каждого OAuth-провайдера
