# Политика безопасности

## Поддерживаемые версии

| Компонент  | Версия | Поддержка |
| ---------- | ------ | --------- |
| apps/web   | latest | Да        |
| apps/admin | latest | Да        |
| apps/api   | latest | Да        |

Поддерживается только последняя версия каждого компонента в ветке `master`.

## Сообщение об уязвимостях

Если вы обнаружили уязвимость безопасности, **не создавайте публичный issue**.

Отправьте отчёт на: **e-nicko@ya.ru**

### Что включить в отчёт

- Описание уязвимости
- Шаги для воспроизведения
- Потенциальное воздействие
- Предлагаемое исправление (при наличии)

### Сроки реагирования

- **Подтверждение получения**: в течение 48 часов
- **Первичная оценка**: в течение 7 дней
- **Исправление критических уязвимостей**: в течение 14 дней

### Процесс

1. Вы отправляете отчёт на e-nicko@ya.ru
2. Мы подтверждаем получение и начинаем анализ
3. Совместно определяем серьёзность и план исправления
4. Выпускаем исправление и уведомляем вас
5. После выпуска исправления вы можете публично раскрыть уязвимость

## Ответственное раскрытие

Мы просим вас:

- Дать нам разумное время на исправление до публичного раскрытия
- Не использовать уязвимость для доступа к чужим данным
- Не нарушать работоспособность сервисов

Мы обязуемся:

- Не предпринимать юридических действий против исследователей, действующих добросовестно
- Работать с вами над пониманием и устранением проблемы
- Упомянуть вас в благодарностях (по вашему желанию)
